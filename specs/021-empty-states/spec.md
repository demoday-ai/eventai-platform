# Feature Specification: Empty States и подсказки

**Feature Branch**: `021-empty-states`
**Created**: 2026-02-09
**Status**: Draft
**Input**: EPIC-012 из admin-panel-usm.md — US-035, US-040, US-041

## User Scenarios & Testing *(mandatory)*

### User Story 1 - Empty States на страницах pipeline-зависимых (Priority: P1)

Организатор открывает страницу, для которой ещё не выполнены предварительные шаги (например, Кластеризация без загруженных проектов, Расписание без одобренной кластеризации). Вместо пустого интерфейса или ошибки он видит понятное объяснение, что нужно сделать, и кнопку-ссылку на нужный шаг.

**Why this priority**: Это ядро фичи — без этого организатор застревает на пустых страницах без понимания, что делать дальше. Касается страниц с жёсткими зависимостями в pipeline.

**Independent Test**: Открыть страницу Кластеризации при пустой БД (без проектов) — должен отображаться empty state с текстом и кнопкой-ссылкой на Импорт.

**Acceptance Scenarios**:

1. **Given** нет загруженных проектов, **When** организатор открывает страницу Кластеризации, **Then** отображается: «Для кластеризации необходимы проекты. Загрузите проекты на странице Импорта» + кнопка «Перейти к импорту» → /import
2. **Given** кластеризация не одобрена, **When** организатор открывает страницу Эксперты (матчинг), **Then** отображается: «Для матчинга экспертов необходима одобренная кластеризация» + кнопка «Перейти к кластеризации» → /clustering
3. **Given** кластеризация не одобрена, **When** организатор открывает страницу Расписание, **Then** отображается: «Для генерации расписания необходима одобренная кластеризация» + кнопка «Перейти к кластеризации» → /clustering
4. **Given** нет загруженных проектов, **When** организатор открывает страницу Проекты, **Then** отображается: «Проекты ещё не загружены. Загрузите проекты на странице Импорта» + кнопка «Перейти к импорту» → /import

### User Story 2 - Empty States на страницах Рассылок (Priority: P2)

Организатор открывает страницу Рассылок при отсутствии события или участников. Отображается понятный empty state, объясняющий, что нужно сделать.

**Why this priority**: Рассылки доступны всегда, но без данных бесполезны. Нужны подсказки для разных сценариев отсутствия данных.

**Independent Test**: Открыть страницу Рассылок при пустой БД — должен отображаться empty state с указанием создать мероприятие.

**Acceptance Scenarios**:

1. **Given** нет активного события, **When** организатор открывает Рассылки, **Then** отображается: «Создайте мероприятие на странице Импорта» + кнопка → /import
2. **Given** событие есть, но нет участников (студенты, эксперты, партнёры = 0), **When** организатор открывает Рассылки, **Then** отображается: «Загрузите участников на странице Импорта» + кнопка → /import
3. **Given** участники есть, но рассылок ещё не было, **When** организатор открывает вкладку «Обзор», **Then** отображается: «Рассылок пока не было. Перейдите на вкладку «Рассылка», чтобы отправить первую»

---

### User Story 3 - Empty States на странице Аудитории (Priority: P3)

Организатор открывает страницу Аудитории до того, как кто-либо взаимодействовал с ботом. Отображается информативный empty state.

**Why this priority**: Аудитория наполняется автоматически через бота, поэтому empty state здесь информационный, а не action-oriented.

**Independent Test**: Открыть страницу Аудитории при пустой БД — должен отображаться empty state с объяснением.

**Acceptance Scenarios**:

1. **Given** нет активного события, **When** организатор открывает Аудиторию, **Then** отображается: «Создайте мероприятие на странице Импорта» + кнопка → /import
2. **Given** событие есть, но нет контактов (guests = 0), **When** организатор открывает Аудиторию, **Then** отображается: «Пока никто не взаимодействовал с ботом. Контакты появятся автоматически, когда участники начнут использовать бота»

---

### Edge Cases

- Данные были, но удалены (например, проекты удалены после кластеризации) → Empty state корректно отображается на зависимых страницах.
- API вернул ошибку, а не пустые данные → Empty state отображается только при пустых данных; ошибки API показываются отдельным error-блоком (уже реализовано).
- Организатор находится на странице, когда данные появляются (auto-refresh) → При следующем auto-refresh empty state исчезает и отображаются данные.
- Частично заполненные данные (например, проекты есть, но студенты нет) → Каждая страница проверяет только свои зависимости, не глобальное состояние.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: Каждая страница админ-панели MUST отображать empty state при отсутствии необходимых данных вместо пустого контента.
- **FR-002**: Empty state MUST содержать: текст-объяснение (что отсутствует) + кнопку-ссылку на шаг, который нужно выполнить.
- **FR-003**: Empty state НЕ MUST блокировать навигацию — все страницы всегда доступны для просмотра.
- **FR-004**: Empty state MUST исчезать автоматически при появлении данных (через существующий auto-refresh).
- **FR-005**: Стиль empty state MUST быть единообразным на всех страницах: карточка с dashed border, иконка Lucide, текст, кнопка-ссылка.
- **FR-006**: Страницы с pipeline-зависимостями (Кластеризация, Эксперты, Расписание) MUST проверять наличие предварительных шагов и показывать empty state если шаг не выполнен.
- **FR-007**: Страница Рассылок MUST показывать разные empty states в зависимости от уровня отсутствия данных: нет события → нет участников → нет рассылок.
- **FR-008**: Страница Аудитории MUST показывать информационный empty state («контакты появятся автоматически»), когда нет контактов.

### Key Entities

- **PageEmptyState**: Переиспользуемый компонент с параметрами: icon, title, description, actionLabel, actionLink. Единый стиль на всех страницах.
- **Pipeline dependencies**: Кластеризация зависит от проектов; Матчинг зависит от одобренной кластеризации; Расписание зависит от одобренной кластеризации.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 100% страниц админ-панели показывают empty state при отсутствии необходимых данных (покрыты все 6+ страниц).
- **SC-002**: Организатор может свободно переходить между всеми разделами при любом состоянии данных — ни одна страница не заблокирована.
- **SC-003**: Каждый empty state на pipeline-зависимой странице содержит actionable кнопку-ссылку на нужный шаг.
- **SC-004**: При появлении данных empty state исчезает без перезагрузки страницы (в течение цикла auto-refresh).
- **SC-005**: Единообразный визуальный стиль empty states на всех страницах (один переиспользуемый компонент).

### Assumptions

- Компонент EmptyState уже существует в проекте и может быть переиспользован или расширен.
- Auto-refresh уже реализован на страницах через TanStack Query refetchInterval.
- Данные о pipeline-зависимостях доступны через существующие API-запросы на страницах.
- Стилистика следует принципам визуала из USM: минимализм, Lucide иконки, функциональный цвет.
