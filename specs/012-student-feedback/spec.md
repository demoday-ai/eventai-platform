# Specification: Student Feedback (EPIC-012)

**Version:** 1.0
**Date:** 2026-02-03
**Dependencies:** EPIC-013 (Expert Scoring)

---

## Problem Statement

После Demo Day студенты хотят получить обратную связь от экспертов, чтобы понять, что было хорошо и что улучшить. Однако:
- Сырые комментарии могут содержать неконструктивную критику ("Херня!")
- Организатору нужно промодерировать ОС перед рассылкой
- ОС должна быть структурирована и полезна студенту

---

## User Stories

### SS-010: Сбор и предобработка ОС
**Как система**, я хочу собрать обратную связь от экспертов и предобработать её, чтобы отфильтровать неконструктив.

**Acceptance Criteria:**
- Сбор текстовых комментариев от экспертов (привязка к проекту)
- AI-фильтрация неконструктивных комментариев
- Группировка по категориям: техническое / продуктовое / презентация

### US-018: Модерация ОС организатором
**Как организатор**, я хочу промодерировать предобработанную ОС перед рассылкой, чтобы до студентов дошёл только полезный фидбэк.

**Acceptance Criteria:**
- Просмотр оригинала + AI-обработки
- Кнопки: Отправить / Редактировать / Отклонить
- Массовая отправка после модерации

### SS-011: Рассылка ОС студентам
**Как система**, я хочу разослать обратную связь каждому студенту, чтобы он получил структурированный фидбэк.

**Acceptance Criteria:**
- Персональное сообщение с ОС
- Группировка: что хорошо / что улучшить / техническое
- Отправка в течение 48 часов после DD

---

## Functional Requirements

### FR-001: Сбор комментариев
- Комментарий привязан к ExpertScore (из EPIC-013)
- Текстовое поле для свободного комментария
- Сохранение в БД

### FR-002: AI-фильтрация
- LLM анализирует комментарий
- Определяет: конструктивный / неконструктивный
- Извлекает ключевые пункты
- Категоризирует: technical / product / presentation / general

### FR-003: Модерация
- Команда /feedback для организатора
- Список проектов с ОС для модерации
- Показ: оригинал + AI-версия
- Действия: approve / edit / reject

### FR-004: Рассылка
- После одобрения — отправка студенту
- Формат: структурированное сообщение
- Статус: sent / pending_moderation / rejected

---

## Data Model

### FeedbackComment (новая сущность)
| Field | Type | Description |
|-------|------|-------------|
| id | UUID | PK |
| project_id | UUID | FK → projects |
| expert_id | UUID | FK → experts |
| original_text | Text | Оригинальный комментарий |
| processed_text | Text | AI-обработанный текст |
| category | Enum | technical, product, presentation, general |
| is_constructive | Boolean | Конструктивный ли комментарий |
| moderation_status | Enum | pending, approved, rejected, edited |
| sent_at | DateTime | Когда отправлен студенту |

---

## Success Criteria

- 90% неконструктивных комментариев отфильтрованы AI
- Модерация одного комментария занимает <30 секунд
- 100% одобренных ОС доставлены студентам
- Студент получает ОС в течение 48 часов после DD

---

## Out of Scope

- Ответы студентов на ОС
- Рейтинг полезности ОС
- Анонимизация экспертов (ОС без указания автора)
