# Feature Specification: Schedule Wizard

**Feature Branch**: `027-schedule-wizard`
**Created**: 2026-02-09
**Status**: Draft
**Input**: User description: "EPIC-007 Расписание: US-022, US-023, US-024"

## User Scenarios & Testing

### User Story 1 - Генерация и просмотр расписания (Priority: P1)

Организатор открывает страницу «Расписание». Если кластеризация одобрена, организатор видит wizard: Генерация → Просмотр → Одобрение. Организатор задаёт параметры (длительность слота, время по залам, перерывы), генерирует расписание и просматривает результат по залам и слотам.

**Why this priority**: Генерация расписания — ключевое действие эпика.

**Independent Test**: При наличии залов запустить генерацию и проверить отображение слотов.

**Acceptance Scenarios**:

1. **Given** кластеризация одобрена, **When** организатор открывает страницу расписания, **Then** отображается wizard с параметрами генерации
2. **Given** wizard на шаге «Генерация», **When** организатор нажимает «Сгенерировать», **Then** расписание создаётся и отображается сетка залы × слоты
3. **Given** расписание отображено, **When** организатор нажимает на слот для редактирования, **Then** открывается inline-форма с полями время, зал, статус

---

### User Story 2 - Одобрение расписания (Priority: P1)

Организатор, удовлетворённый расписанием, одобряет его. При нажатии «Одобрить» появляется диалог подтверждения. После подтверждения расписание фиксируется, отображается подсказка о следующем шаге (настройка напоминаний), глобальный прогресс обновляется.

**Why this priority**: Одобрение — критический шаг, завершающий фазу «Распределение».

**Acceptance Scenarios**:

1. **Given** расписание сгенерировано, **When** организатор нажимает «Одобрить», **Then** появляется диалог подтверждения
2. **Given** диалог подтверждения открыт, **When** организатор нажимает «Подтвердить», **Then** расписание одобрено, отображается подсказка «Расписание одобрено. Можно отправить напоминания» со ссылкой на /reminders
3. **Given** расписание одобрено, **When** организатор просматривает глобальный прогресс, **Then** фаза «Распределение» отмечена как завершённая

---

### Edge Cases

- Нет одобренной кластеризации → empty state с подсказкой
- Пустое расписание → сообщение «Расписание пусто»

## Requirements

### Functional Requirements

- **FR-001**: Система ДОЛЖНА показывать диалог подтверждения при одобрении расписания
- **FR-002**: После одобрения система ДОЛЖНА показывать подсказку со ссылкой на следующий шаг (напоминания)
- **FR-003**: После одобрения система ДОЛЖНА обновлять глобальный прогресс

## Success Criteria

### Measurable Outcomes

- **SC-001**: Организатор может одобрить расписание за 3 клика
- **SC-002**: После одобрения глобальный прогресс обновляется без перезагрузки
- **SC-003**: Все страницы без данных показывают empty state с подсказкой
