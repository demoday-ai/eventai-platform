# Specification: Organizer Dashboard (EPIC-011)

**Version:** 1.0
**Date:** 2026-02-03
**Dependencies:** EPIC-003 (Confirmations), EPIC-006 (Coverage)

---

## Problem Statement

Во время Demo Day организаторам нужно в реальном времени видеть статус мероприятия:
- Кто из студентов пришёл, кто не явился
- Какие залы обеспечены экспертами, какие нет
- Есть ли пустые слоты (отменённые выступления)
- Сколько гостей пришло и какого типа

Без этой информации сложно оперативно реагировать на проблемы.

---

## User Stories

### US-017: Статус DD в реальном времени
**Как организатор**, я хочу видеть статус Demo Day в реальном времени, чтобы быстро реагировать на проблемы.

**Acceptance Criteria:**
- Команда `/dashboard` показывает сводку по всем метрикам
- Информация обновляется при каждом запросе (pull-модель)
- Критические проблемы выделены (нет эксперта, много неявок)

---

## Functional Requirements

### FR-001: Команда /dashboard
- Доступна только организаторам
- Показывает сводку по текущему событию
- Обновляется при каждом вызове

### FR-002: Метрики студентов
- Всего: N проектов
- Подтвердили: N (% от общего)
- Пришли (checkin): N (% от подтвердивших)
- Не явились: N
- Список неявившихся с контактами

### FR-003: Метрики экспертов
- Всего приглашено: N
- Подтвердили: N
- По залам: покрытие (подтверждённые / нужно)
- Залы без экспертов (алерт)

### FR-004: Метрики гостей
- Всего зарегистрировано: N
- По подтипам: Абитуриенты / AI-практики / Другое / Бизнес-партнёры
- Прошли профилирование: N

### FR-005: Алерты
- Критические: залы без экспертов, >20% неявок студентов
- Предупреждения: слоты близко к началу без подтверждения
- Выводятся в начале дашборда

### FR-006: Детализация по кнопкам
- "Неявившиеся студенты" — список с контактами
- "Проблемные залы" — детали по залам без экспертов
- "Гости" — распределение по типам

## Data Requirements

### Существующие данные
- ParticipationRequest (подтверждения студентов)
- ExpertRoomAssignment (распределение экспертов)
- User с guest_subtype (гости)
- BusinessProfile (бизнес-партнёры)

### Новые данные
- Нет новых таблиц (используем существующие)
- Чекин студентов — через существующий participation status

---

## Success Criteria

- Дашборд загружается за <3 секунды
- Информация актуальна на момент запроса
- Критические алерты видны сразу
- 100% организаторов могут использовать без инструкции

---

## Out of Scope

- Push-уведомления при изменениях (только pull)
- Веб-интерфейс (только Telegram)
- Исторические данные и графики
- Экспорт в Excel/PDF

---

## Assumptions

- Организаторы имеют Telegram и настроенный бот
- Данные о подтверждениях актуальны в БД
- Check-in реализован через participation status
