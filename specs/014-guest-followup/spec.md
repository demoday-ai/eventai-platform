# Specification: Guest Follow-up (EPIC-014)

**Version:** 1.0
**Date:** 2026-02-03
**Dependencies:** EPIC-005 (Guest Profiling), EPIC-009 (Q&A Helper)

---

## Problem Statement

После Demo Day гости не получают структурированный follow-up:
- Забывают детали проектов через несколько дней
- Нет контактов команд (или нужно искать самим)
- Нет единого места с заметками и впечатлениями

---

## User Stories

### US-020: Follow-up пакет для гостя
**Как гость**, я хочу получить follow-up пакет после Demo Day, чтобы вернуться к понравившимся проектам.

**Acceptance Criteria:**
- Пакет автоматически формируется через 24ч после DD
- Включает: список посещённых проектов, мои оценки, контакты (если разрешено)
- Команда /followup для ручного запроса
- Пакет в читаемом формате (Telegram + опционально PDF)

---

## Functional Requirements

### FR-001: Автоматическая генерация
- Через 24 часа после окончания DD отправить всем гостям
- Сформировать персональный пакет по профилю

### FR-002: Содержимое пакета
- Список проектов, которые гость отметил (рекомендации)
- Оценки и комментарии гостя (если давал)
- Контакты проектов (Telegram, если студент дал согласие)
- Шаблон письма для связи

### FR-003: Ручной запрос
- Команда /followup — получить пакет в любой момент
- Обновляется при повторном запросе

### FR-004: Контроль доступа к контактам
- Контакт студента показывается только если:
  - Студент дал согласие (contact_allowed)
  - Гость создал ContactRequest и он approved

---

## Data Model

### FollowupPackage (новая сущность)
| Field | Type | Description |
|-------|------|-------------|
| id | UUID | PK |
| guest_id | UUID | FK → users (guest) |
| event_id | UUID | FK → events |
| generated_at | DateTime | Время генерации |
| content | JSON | Кэш содержимого пакета |
| sent | Boolean | Отправлен ли |

---

## Success Criteria

- 80% гостей получают follow-up в течение 48ч
- Пакет содержит контакты минимум 3 проектов
- Время генерации пакета <5 секунд
