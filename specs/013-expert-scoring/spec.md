# Specification: Expert Scoring (EPIC-013)

**Version:** 1.0
**Date:** 2026-02-03
**Dependencies:** EPIC-004 (Expert Assignment), EPIC-012 (Feedback)

---

## Problem Statement

Эксперты на Demo Day оценивают проекты, но текущий процесс неудобен:
- Путаница в Google Таблицах при пропуске студентов
- Забываются впечатления к концу дня
- Нет структурированной оценки по критериям

---

## User Stories

### US-019: Оценка проекта экспертом
**Как эксперт**, я хочу быстро оценить проекты прямо в Telegram, чтобы не забыть впечатления.

**Acceptance Criteria:**
- Бот присылает форму оценки после каждого проекта
- Оценка по критериям (не просто 1-5): Актуальность, Новизна, Реализация, и др.
- Комментарий в свободной форме (опционально)
- Автотрекинг пропущенных (если студент не выступил — не спрашивать)
- Занимает <2 мин на проект
- Напоминание через день, если не оценил

---

## Functional Requirements

### FR-001: Триггер оценки
- Бот отправляет форму эксперту после завершения слота
- Привязка к schedule_slot (проект → время)
- Можно пропустить ("Студент не выступил")

### FR-002: Критерии оценки
- Актуальность (1-3)
- Практическая значимость (1-3)
- Новизна/инновационность (1-3)
- Качество реализации (1-3)
- Масштабируемость (1-3)
- R&D компонент (1-3)
- Общее впечатление (1-5)

### FR-003: Комментарий
- Текстовое поле (опционально)
- Сохраняется в FeedbackComment для EPIC-012

### FR-004: Напоминания
- Если эксперт не оценил за 24 часа — напоминание
- Список неоценённых проектов

### FR-005: Ручной запуск
- Команда /score для просмотра неоценённых
- Выбор проекта и оценка

## Data Model

### ExpertScore (новая сущность)
| Field | Type | Description |
|-------|------|-------------|
| id | UUID | PK |
| expert_id | UUID | FK → experts |
| project_id | UUID | FK → projects |
| slot_id | UUID | FK → schedule_slots (опционально) |
| relevance | Int | Актуальность (1-3) |
| practical_value | Int | Практическая значимость (1-3) |
| novelty | Int | Новизна (1-3) |
| implementation | Int | Качество реализации (1-3) |
| scalability | Int | Масштабируемость (1-3) |
| research | Int | R&D компонент (1-3) |
| overall | Int | Общее (1-5) |
| skipped | Boolean | Студент не выступил |

---

## Success Criteria

- Оценка занимает <2 минут
- 90% экспертов оценивают проекты через бот
- 0 путаницы с нумерацией проектов
