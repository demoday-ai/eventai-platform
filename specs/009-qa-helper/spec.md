# Specification: Q&A Helper (EPIC-009)

**Version:** 1.0
**Date:** 2026-02-03
**Dependencies:** EPIC-005 (Guest Profiling), EPIC-006 (Business Profiling)

---

## Problem Statement

Гости и бизнес-партнёры Demo Day приходят смотреть проекты, но часто не знают, какие вопросы задавать во время Q&A-сессий. Это приводит к:
- Поверхностным вопросам ("Что вы делаете?")
- Неэффективному использованию времени Q&A (15 мин на проект)
- Упущенной возможности глубже понять проект

Эксперты **не нуждаются** в подсказках вопросов — по результатам CustDev интервью #2: "Вопросы от человека."

---

## User Stories

### US-012: Вопросы для гостей
**Как гость**, я хочу получить предложенные вопросы для проектов из моей программы, чтобы задать содержательные вопросы на Q&A.

**Acceptance Criteria:**
- По каждому проекту из персональной программы — 3-5 вопросов
- Вопросы адаптированы под подтип гостя:
  - Абитуриент: фокус на обучении, команде, технологиях для изучения
  - AI-практик: фокус на архитектуре, подходах, метриках, воспроизводимости
  - Другое: универсальные вопросы про проект и его ценность
- Вопросы учитывают профиль интересов гостя (теги, тематики)

### US-013: Бизнес-вопросы для партнёров
**Как бизнес-партнёр**, я хочу получить вопросы с бизнес-фокусом, чтобы оценить партнёрский потенциал проекта.

**Acceptance Criteria:**
- По каждому проекту из подборки — 3-5 бизнес-вопросов
- Вопросы зависят от задачи партнёра:
  - Инвестор: unit-экономика, рынок, команда, раунд
  - HR/нанимающий: стек, опыт, готовность к работе, интересы
  - Технологический партнёр: интеграция, API, масштабирование
  - Партнёрство: бизнес-модель, готовность к пилоту, условия

### US-014: Матрица сравнения проектов
**Как гость или бизнес-партнёр**, я хочу увидеть матрицу сравнения проектов, чтобы сопоставить их по нужным критериям.

**Acceptance Criteria:**
- Формат: таблица проектов × критериев
- Критерии формируются автоматически из профиля:
  - Гость: тематика, стек, стадия, применимость
  - Бизнес: бизнес-модель, стадия, рынок, команда
- Возможность добавить кастомный критерий
- Сортировка по любому критерию

## Functional Requirements

### FR-001: Генерация вопросов
- Система генерирует вопросы через LLM (OpenRouter API)
- Промпт включает: описание проекта, профиль пользователя, подтип/задачу
- Вопросы возвращаются в структурированном формате (JSON)
- Кэширование вопросов на 1 час (избежание повторных запросов к LLM)

### FR-002: Команда /questions
- Доступна ролям: Гость, Бизнес/партнёр
- Показывает список проектов из персональной программы
- При выборе проекта — показывает 3-5 вопросов
- Inline-кнопки: "Ещё вопросы" / "Другой проект" / "Назад"

### FR-003: Матрица сравнения
- Команда /compare
- Выбор проектов для сравнения (2-5)
- Автоматическая генерация критериев из профиля
- Текстовый вывод в формате таблицы (Markdown)
- Кнопка "Добавить критерий" — ввод свободного текста

### FR-004: Интеграция с рекомендациями
- Вопросы привязаны к проектам из ProjectRecommendation
- При отсутствии рекомендаций — предложение пройти профилирование

## Data Model

### QASuggestion (новая сущность)
| Field | Type | Description |
|-------|------|-------------|
| id | UUID | PK |
| user_id | UUID | FK → users |
| project_id | UUID | FK → projects |
| questions | JSON | Список сгенерированных вопросов |
| question_type | Enum | guest_general, guest_applicant, guest_practitioner, business_investor, business_hr, business_partner, business_tech |
| generated_at | DateTime | Время генерации |
| expires_at | DateTime | TTL кэша (1 час) |

## User Scenarios

### Scenario 1: Гость запрашивает вопросы
1. Гость (уже профилирован) отправляет /questions
2. Бот показывает список проектов из программы (inline-кнопки)
3. Гость выбирает проект
4. Бот генерирует 3-5 вопросов под подтип гостя
5. Гость видит вопросы, может запросить ещё или выбрать другой проект

### Scenario 2: Бизнес-партнёр запрашивает вопросы
1. Бизнес-партнёр отправляет /questions
2. Бот показывает проекты из бизнес-подборки
3. Партнёр выбирает проект
4. Бот генерирует бизнес-вопросы на основе задачи (инвестор/HR/партнёр)
5. Вопросы фокусируются на бизнес-метриках

### Scenario 3: Матрица сравнения
1. Гость/партнёр отправляет /compare
2. Бот показывает проекты для выбора (checkbox-стиль)
3. Пользователь выбирает 2-5 проектов и подтверждает
4. Бот генерирует матрицу с автокритериями
5. Пользователь может добавить свой критерий

---

## Success Criteria

- Гости и бизнес-партнёры получают релевантные вопросы за <5 секунд
- 80% вопросов оцениваются пользователями как "полезные"
- Матрица сравнения формируется за <10 секунд
- Система обрабатывает 50 одновременных запросов без деградации

---

## Out of Scope

- Вопросы для экспертов (CustDev: "Вопросы от человека")
- Автоматическая оценка ответов студентов
- Голосовой ввод вопросов
- Интеграция с видеозаписями Q&A

---

## Assumptions

- У пользователя уже есть персональная программа (профилирование пройдено)
- OpenRouter API доступен и стабилен
- Описания проектов достаточно информативны для генерации вопросов
