## team00 — Michil Deploy Workshop

### PulseTrack – Fitness Tracking Landing

PulseTrack is a fitness tracking landing page built with React, TypeScript, Vite, and Radix Themes. It showcases a training analytics app with a marketing home page and an About page featuring a contact form.

**Tech stack**

- React + TypeScript + Vite
- `@radix-ui/themes` and `@radix-ui/react-icons`
- `react-router-dom`
- `react-hook-form` (built-in rules only, no schema validators)
- `@tanstack/react-query`

**Getting started**

1. Install dependencies: `npm install`
2. Run the dev server: `npm run dev`
3. Open the app in your browser (usually at `http://localhost:5173`).

**Available scripts**

- `npm run dev` – start Vite dev server
- `npm run build` – type-check and build for production
- `npm run preview` – preview the production build
- `npm run lint` – run ESLint

**Routes**

- `/` – **Home**: hero section, highlights, features grid, "How it works" steps, and final CTA, all styled with Radix Themes.
- `/about` – **About & Contact**: about copy plus a contact form with validation and submit handled via React Hook Form and React Query.

**Contact form & draft persistence**

- Validation is implemented via `react-hook-form` using `register` rules:
  - `name`: required, min length 2
  - `email`: required, email pattern
  - `topic`: required select
  - `message`: required, min length 20
  - `consent`: required checkbox with custom `validate`
- Draft values are stored in `localStorage` under the `contactDraft` key and restored on page load; resetting or successful submit clears the draft.

Evaluates papers across six dimensions: Code Availability, Data Availability, Infrastructure/Compute Requirements, Tooling & Dependencies, Reproducibility Evidence, and Documentation Quality. **Decision Logic:** NO-GO when critical categories show BLOCKER status, CAUTIOUS for multiple RISK signals, GO otherwise.

## team03 — Edward

### Edward — AI Process Optimizer

AI-система для автоматизации интервью и оптимизации бизнес-процессов.

**Проблема:** Бизнес-консультанты тратят дни/недели на интервью, ручную транскрипцию, анализ и картирование процессов. 60-70% времени уходит на транскрипцию/анализ вместо поиска решений. Неполная видимость процессов — каждый участник видит только свою часть.

**Решение:** Edward анализирует интервью с разными участниками процесса, консолидирует перспективы в карту AS IS, генерирует оптимизированную карту TO BE и создаёт структурированные отчёты с рекомендациями — "за часы вместо недель."

**Ключевые возможности:**
- Анализ интервью по 8 профессиональным методикам (5 Whys, VSM, JTBD, Process Mining, Lean Six Sigma, BPMN, RACI, BABOK)
- Консолидация перспектив: объединение интервью разных участников с подсветкой различий
- Автогенерация карт процессов AS IS и TO BE (Mermaid)
- Структурированные отчёты: pain points, bottlenecks, рекомендации по оптимизации, возможности внедрения AI, RACI-матрица
- Интервью в реальном времени: AI генерирует контекстные вопросы live
- Экспорт: PNG/SVG карты, PDF отчёты

**Стек:**
- Frontend: React 19.2 + TypeScript 5.9, Vite, Tailwind CSS, shadcn/ui
- Backend: Python 3.11+, FastAPI
- БД: PostgreSQL 15+, Qdrant (vector), Neo4j (graph)
- LLM: OpenRouter SDK
- Speech: Whisper API
- Task Queue: Celery + Redis
- Rendering: Mermaid.js, WeasyPrint

**Быстрый старт:** `make setup && make dev`

## team14 — AI Label

### AI Label — Сервис анализа вирусного контента в коротких видео

Находит растущие тренды до насыщения, анализирует что работает и почему, даёт готовый сценарий — иди снимай.

**Стадия:** Discovery → MVP

**Проблема:** Начинающие авторы коротких видео тратят часы на скроллинг в поисках идей, узнают о трендах слишком поздно и не могут адаптировать их под себя.

**Решение:** Единый инструмент: метадата-аналитика (звуки, хештеги, тренды) + визуальный AI-анализ (хуки, формат, монтаж) + анализ аудитории (комментарии, bait detection, sentiment) + генерация персонализированных сценариев.

**Прототип (CLI) — 4 режима:**
```
uv run main.py trending --count 10       # Анализ трендов FYP
uv run main.py profile @username          # Анализ профиля
uv run main.py hashtag cooking --count 30 # Анализ ниши
uv run main.py sound 7016547803243022337  # Анализ звука
```

Опции: `--with-video` (визуальный анализ Nemotron VL), `--with-comments` (анализ комментариев GPT-4.1-mini), `--days N`, `--count N`.

**Что анализируется:**
- Метадата: Engagement Rate, топ звуки/хештеги, лучшие часы, частота постинга, CTA, trending sounds
- Визуальный анализ: текст на экране, формат (talking head, split screen...), количество сцен, качество
- Комментарии: rage bait / comment bait detection, sentiment, controversy score, red flags

## team17 — Опора

### Unwind — Ваша голова — плохой предсказатель

> Ваша голова — плохой предсказатель, который любит тревожить. И чтобы перестать жить по этим правилам мы с командой сделали под это мини-приложение

[Try in Telegram](https://t.me/AICampUnwindBot)

**Проблема:**
- Тревожные мысли крутятся в голове и кажутся реальными
- Человек не видит, что большинство предсказаний не сбывается
- Нет простого инструмента отследить «а что было на самом деле?»

> Факт: в исследованиях Penn State 91% тревожных предсказаний не сбылись.

**Решение:**
- Voice-first — записываешь голосом, не нужно формулировать текст
- AI-разбор — разделение на факты и предсказания
- Follow-up — через время спросим «как прошло?»
- Статистика — по категориям: работа, здоровье, отношения, деньги, учёба

**Для кого:** Тревожные люди (мягкость, голосовой ввод, без давления). Self-trackers (данные, паттерны, статистика).

**Преимущества:** Voice-first (низкий порог входа), Streak (никогда не сбрасывается), Telegram (уже у всех).

**Философия:**
- Друг с блокнотом, не терапевт
- Ненасильственное общение: наблюдения вместо интерпретаций
- Экстернализация: «Голова предсказала» — не «ты боялся»
- Streak без наказания

**Стек:** React 19, TypeScript, Vite, Tailwind CSS 4, @telegram-apps/sdk-react (Mini App), Shadcn/ui, Lucide React, Docker, nginx, Let's Encrypt, GitHub Actions.

**Команда:** 3 разработчика, 5 дней на MVP.
