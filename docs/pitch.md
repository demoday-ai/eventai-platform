# EventAI — Питч Demo Day

**Формат:** 7 минут + вопросы

---

## One-liner

Telegram-бот для **организаторов конференций**, который даёт каждому гостю персональную программу **за минуту** AI-диалога.

---

## Проблема и ICP

### ICP: Организаторы Demo Day, конференций, хакатонов

**Боль организаторов:**
- Расписание вручную за ночь до DD
- 80 студентов не попали в расписание
- Нужно 60 экспертов, активных — 30
- Нет аналитики и follow-up

**Боль гостей (партнёры, HR, инвесторы):**
- 330 проектов в 10 залах
- Видят <20% контента
- Пропускают 68% релевантного
- Нет персонализации

**Цена боли:** организатор тратит 8+ часов на расписание, гость уходит разочарованным, бизнес-партнёр не находит нужный проект → потерянные контакты и сделки.

---

## Решение (и почему нельзя просто спросить у ChatGPT)

1. **Нет данных** — ChatGPT не знает ваши 330 проектов, залы, расписание
2. **Нет действий** — не может отправить напоминание, записать на слот, запросить контакт
3. **Нет стабильности** — каждый запрос — новый контекст, нет памяти профиля гостя
4. **Нет контроля** — организатор не видит аналитику, не управляет контентом

**EventAI** = специализированный агент с данными события + действиями + памятью + админкой.

## Технология

**Где AI даёт ценность:**

- **TF-IDF** — быстрый отбор по тегам
- **LLM re-ranking** — Claude/GPT переранжирует топ-20 с учётом контекста профиля
- **Tool-calling** — агент вызывает инструменты (поиск, Q&A, сравнение)

**Ключевое:** LLM понимает *почему* гостю интересен проект (контекст, задачи), а не просто совпадение тегов.

**Стек:**
- python-telegram-bot 21.x
- FastAPI + PostgreSQL
- OpenRouter (Claude 3.5, GPT-4o)
- React 19 + TanStack Query

## Evaluation

**Методология:** 10 профилей × 330 проектов × экспертная разметка (0-3 релевантность)

| Метрика | Значение | Описание |
|---------|----------|----------|
| NDCG@15 | 0.82 | Релевантные проекты в топе |
| Precision@15 | 0.71 | 7 из 10 в цель |
| Recall@15 | 0.78 | Находим 8 из 10 подходящих |

**Сильные стороны:** точный топ-5, хорошо работает на чётких интересах (NLP, HR).

**Слабые места:** размытые запросы ("что-нибудь интересное"), редкие теги.

## Гипотезы и пивоты

| Гипотеза | Статус | Вывод |
|----------|--------|-------|
| Гости хотят персонализацию | ✓ | Tinder-механика медленная, топ с рейтингом — быстрее |
| Организаторам нужна автоматизация | ✓ | Расписание вручную — главная боль |
| Q&A нужен экспертам | ✗ | "Эксперт сам знает, что спросить". Q&A — для гостей |
| 5 ролей = 5 разных UX | ✓ | Разные потребности у каждой роли |

**Главный пивот:** фокус сместился с гостей на организаторов (автокластеризация, расписание).

## Экономика

**Модель:** B2B — организаторы платят за событие

| Параметр | Значение |
|----------|----------|
| Цена за событие | от 100 000 ₽ |
| Cost на гостя | ~5-10 ₽ (LLM API) |
| Cost на событие | ~3-5K ₽ |
| Маржа | >90% |

**Снижение cost:** кэширование, локальные модели, batch-запросы.
