> **Статус:** Концепт для обсуждения и имплементации

## Проблема текущих решений

### Demo-Hero (существующий бот JUGru)

❌ **Поверхностное профилирование:**
- "Пол" + "Чем занимаешься?" — недостаточно для 330 проектов
- Не понимает задачу гостя (найти партнёров? нанять? инвестировать?)
- Не учитывает роль (инвестор vs HR vs студент)

❌ **Медленная Tinder-механика:**
- Показывает пары проектов → выбираешь один
- На 330 проектов нужно >100 свайпов
- Нет контекста, почему именно эти проекты в паре
- Невозможно отфильтровать ("покажи только NLP")

❌ **Баги и потеря данных:**
- Маршрут генерируется с ошибками
- Пользователь выбирает проекты → маршрут пустой
- Нет персистентности через рестарт бота

❌ **Нет помощи во время DD:**
- Нет Q&A подсказок для гостей
- Нет брифинга перед проектом
- Нет матрицы сравнения проектов

❌ **Нет follow-up:**
- После DD — только оценка 1-5
- Нет контактов авторов
- Нет конспекта программы
- Нет шаблонов писем / LOI

### Боль целевой персоны (Марина, PM в FinTech)

> "Я пришла, посмотрела 3 проекта наугад, ничего не поняла и ушла. Максимально неудобное мероприятие."

**Ключевые проблемы:**
- 330 проектов в 6-10 залах → невозможно сориентироваться
- Физически успевает посетить <20% проектов (34 из 212)
- Выбирает наугад → попадает на нерелевантные → теряет время
- Нет follow-up → контакты и впечатления теряются через 2 дня
- Низкая доходимость: из 190 регистраций ~50 приходят реально

## Принципы идеального UX

### 1. **Быстрый онбординг (2-3 мин до программы)**

Гость не должен тратить час на свайпы. Профилирование — разговор с агентом 2-3 минуты → персональная программа готова.

**Метрика:** 80% пользователей получают программу за <3 минуты после /start.

### 2. **Разговорный интерфейс (агент, не форма)**

Бот — это AI-агент, а не форма с полями. Пользователь пишет "Я ищу NLP-проекты для финтеха" → агент понимает и формирует подборку. Не нужно выбирать из 50 тегов.

**Метрика:** 90% пользователей используют текстовые ответы, а не только кнопки.

### 3. **Контекстное понимание роли и задач**

Бот знает, кто перед ним (инвестор / HR / студент), и подстраивает вопросы:
- **Инвестор:** "Какая стадия? Pre-seed / Seed?"
- **HR:** "Какой стек ищете? Какой уровень?"
- **Студент:** "Когда выступаешь? Нужна обратная связь?"

**Метрика:** У каждой роли свой сценарий профилирования (5 ролей × 3-5 вопросов).

### 4. **Проактивность (бот предлагает следующие шаги)**

Не ждёт команды от пользователя, а сам:
- Напоминает за день / за час о программе
- Уведомляет о сдвигах тайминга
- Предлагает Q&A вопросы перед проектом
- Отправляет follow-up пакет после DD

**Метрика:** 80% пользователей получают как минимум 1 проактивное сообщение.

### 5. **Непрерывный journey (до → во время → после DD)**

Бот сопровождает гостя на всём пути:
- **До DD:** профилирование → программа → брифинг
- **Во время DD:** напоминания → Q&A помощь → агент-режим
- **После DD:** обратная связь → контакты → LOI

**Метрика:** Engagement rate: 60% пользователей взаимодействуют с ботом на 2+ этапах.

### 6. **Ясность и доверие (нет чёрного ящика)**

Пользователь всегда понимает, почему бот предлагает именно эти проекты:
- "По вашим интересам (NLP, FinTech) — 12 проектов"
- "Конфликт: Проект A и B идут одновременно в разных залах"
- "Совет: Проект C высоко оценён экспертами (2.8/3.0)"

**Метрика:** 70% пользователей оценивают рекомендации как релевантные (NPS 7+).

## Концепция по этапам

### Этап 1: Онбординг (30 сек)

**Цель:** Определить роль и создать первый контакт.

**Flow:**

```
Пользователь: /start

Бот:
Привет! Я AI-куратор Demo Day. Помогу найти интересные проекты,
спланировать программу и организовать встречи с авторами.

Кто вы?

[Кнопки: Гость | Бизнес-партнёр | Студент | Эксперт | Организатор]
```

**Для гостя — подтип (опционально):**

```
Бот: Расскажите чуть больше — это поможет мне подобрать программу:

[Кнопки: Абитуриент | AI-практик | Другое | Пропустить]
```

**Ключевое:**
- Один клик → роль выбрана
- Подтип — опционально, для аналитики (не блокирует продолжение)
- Сразу переходим к профилированию (нет промежуточных экранов)

---

### Этап 2: Профилирование (2-3 мин)

**Цель:** Понять задачи и интересы пользователя через разговор с агентом.

**Не Demo-Hero Tinder!** Вместо этого — короткий диалог:

```
Бот:
Отлично! На Demo Day будет ~330 проектов в 6-10 залах.
Я помогу найти самое интересное для вас.

Расскажите, что ищете? Например:
• "Проекты по NLP и языковым моделям"
• "Стартапы в FinTech, готовые к инвестициям"
• "Инженеров со знанием Python + ML для найма"

[Textarea для текстового ответа]
```

**Пользователь:**

```
Я ищу проекты по NLP и автономным агентам для финтеха.
Интересуют готовые решения для интеграции или R&D-коллаборации.
```

**Агент анализирует → уточняющие вопросы (опционально):**

```
Бот:
Отлично! Уточню пару моментов:

1. Стадия проектов: прототип, MVP или production-ready?
   [Кнопки: Прототип | MVP | Production | Не важно]

2. Стек: есть ли предпочтения по моделям?
   [Кнопки: GPT-4 | Claude | Open-source LLM | Не важно]

3. Формат: интересуют только коммерческие проекты или любые?
   [Кнопки: Коммерческие | Исследовательские | Любые]
```

**Ключевое:**
- **Текст + кнопки:** Пользователь может написать свободным текстом ИЛИ выбрать из вариантов
- **Агент адаптируется:** Если написал подробно → меньше уточнений. Если кратко → больше вопросов
- **Максимум 3-5 вопросов** (не 100 свайпов как в Tinder)
- **LLM-powered:** агент понимает контекст (NLP + FinTech = semantic search по проектам)

**Подтверждение профиля:**

```
Бот:
Отлично! Вот что я понял:

✅ Тематики: NLP, Автономные агенты
✅ Индустрия: FinTech
✅ Стадия: MVP или production-ready
✅ Формат: коммерческие + R&D-коллаборации

Всё верно?

[Кнопки: Да, подбери программу | Изменить | Начать заново]
```

   NLP, Conversational AI, Banking
   → Виртуальный ассистент для клиентов банка.

📝 Напишите мне:
• номер — подробности о проекте
• "сравни 1 и 2" — матрица сравнения

• "хочу другое" — изменю подборку
```

**Дополнительно (если осталось время):**

```
Бот:
📚 Дополнительно (если успеете):

4. **NLP для анализа новостей и прогноза курсов** — 75%

**Ключевое:**
- **Must-visit (top-N):** 10-15 проектов с высокой релевантностью
- **If-time:** ещё 5-10 проектов (lower priority)
- **Конфликты:** явно показаны (одновременно в разных залах)
- **Краткие брифы:** название + зал + время + теги + summary (1-2 предложения)
- **Экшены:** пользователь может сразу углубиться ("номер 3") или сравнить ("сравни 1 и 2")

## Ключевые отличия от Demo-Hero

| Параметр | Demo-Hero | AI-куратор (наш) |
|----------|-----------|------------------|
| **Профилирование** | Пол + "чем занимаешься" | Роль + задачи + отрасль/стек (агент-диалог) |
| **Подборка проектов** | Tinder-пары (>100 свайпов) | Мгновенная подборка (12 must-visit за 5 сек) |
| **Время онбординга** | 30-60 мин (Tinder) | 2-3 мин (диалог с агентом) |
| **Контекстность** | Бинарный выбор (A vs B) | Семантический поиск (NLP + FinTech + MVP) |
| **Во время DD** | Только напоминания | Q&A помощь + матрица сравнения + агент-режим |
| **Follow-up** | Оценка 1-5 | Конспект + контакты + шаблоны писем + next steps |
| **Персистентность** | Баги (маршрут теряется) | PicklePersistence (state сохраняется) |
| **Проактивность** | Минимальная | Напоминания + брифинг + уведомления о сдвигах |

**Ключевое преимущество:** AI-куратор — это не форма, а умный помощник, который:
- Понимает контекст (роль + задачи)
- Быстро формирует релевантную программу (5 сек vs 60 мин)
- Помогает во время DD (Q&A, сравнение, брифинг)
- Не даёт впечатлениям потеряться (follow-up пакет)

## Метрики успеха UX

### Онбординг
- **Time to first program:** 80% пользователей получают программу за <3 мин
- **Completion rate:** 70% пользователей завершают профилирование

## Технические требования к имплементации

### Backend

1. **LLM-powered профилирование:**
   - Разговорный агент (не форма): пользователь пишет текстом → LLM анализирует
   - Извлечение тегов, отрасли, стадии из свободного текста
   - Адаптивные уточняющие вопросы (если неясно → спрашивает)

2. **Семантический поиск проектов:**
   - Эмбеддинги проектов (title + description + tags)
   - Semantic search по запросу пользователя (cosine similarity)
   - Ранжирование: relevance score + expert ratings + availability (время/зал)

3. **Конфликт-детекция:**
   - Проекты в разных залах одновременно → явно показываем
   - Предлагаем альтернативы (if-time список)

4. **Q&A генерация:**
   - LLM генерирует вопросы на основе профиля гостя + описания проекта
   - Разбивка: технические / бизнес / R&D

5. **Матрица сравнения:**
   - LLM сравнивает проекты по критериям (стадия, стек, задача, контакт)
   - Совет: какой проект более зрелый / релевантный

6. **Follow-up генерация:**
   - Шаблоны писем: коллаборация / найм / инвестиции
   - Персонализация: имя гостя, компания, проект
   - Next steps: рекомендации на основе поведения (какие проекты посетил)

### Frontend (бот)

1. **State persistence:**
   - PicklePersistence: conversation state сохраняется через рестарты
   - User profile: роль, подтип, теги, selected projects

2. **Markdown formatting:**
   - Рич-карточки проектов: bold, italic, bullet lists
   - Escape special chars (не ломать форматирование)

3. **Inline keyboards:**
   - Роли, подтипы, подтверждения
   - Quick actions: "Вопросы к проекту", "Сравни", "Записаться"

4. **Long messages:**
   - Must-visit список может быть длинным → split по 4096 chars
   - Truncation: summary по 120 chars, не по середине слова

5. **Proactive messages:**
   - Напоминания за 15 мин (scheduler + Telegram send)
   - Уведомления о сдвигах тайминга (если расписание изменилось)

80% гостей получают релевантную программу за <3 мин и взаимодействуют с ботом на 2+ этапах. NPS 40+.

**Ключевое отличие от Demo-Hero:**
Не форма, не Tinder, а умный помощник, который понимает контекст и ведёт гостя от онбординга до follow-up.
