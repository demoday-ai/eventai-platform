## Новые и расширенные истории

### [EPIC-002+] Загрузка и кластеризация (расширение)

---
**[US-025][Организатор] Как организатор, я хочу задать параметры кластеризации (макс. проектов в кластере, перерывы, лимит по времени), чтобы получить кластеры под формат мероприятия.**
- [ ] Поле "Макс. проектов в кластере" (по умолчанию 5)
- [ ] Поле "Длительность перерыва" (по умолчанию 15 мин)
- [ ] Поле "Ограничение по времени зала" (напр. 10:30-19:30)
- [ ] 5 примеров разного типа в подсказке
- [ ] Параметры передаются в LLM-промпт при кластеризации
- Notes: Инна: "хочу каждый кластер до 5 проектов, потом перерыв"

---
**[US-026][Организатор] Как организатор, я хочу пометить проекты как "для бизнеса" или "для прожарки", чтобы они попали в одну комнату независимо от тематики.**
- [ ] Тег/флаг "Формат защиты": обычный / бизнес / прожарка
- [ ] Проекты с одинаковым форматом группируются в один зал
- [ ] Формат имеет приоритет над тематическим совпадением
- Notes: Инна: "у них могут быть совсем разные тематики, но их нужно объединить вместе"

---
**[US-027][Организатор] Как организатор, я хочу получить 2-3 варианта кластеризации на выбор, чтобы сравнить и утвердить лучший.**
- [ ] Кнопка "Сгенерировать еще вариант" (до 3 вариантов)
- [ ] Каждый вариант: список залов с проектами + обоснование
- [ ] Сравнение вариантов: кол-во залов, баланс, тематическая когерентность
- [ ] Выбор лучшего -> утверждение
- [ ] Прогресс-бар (генерация 1 варианта ~2 мин)
- Notes: Инна: "по-хорошему сделать пару вариантов кластеризации"

---
**[US-028][Организатор] Как организатор, я хочу добавлять студентов и редактировать названия проектов, чтобы исправить ошибки без перезагрузки данных.**
- [ ] Кнопка "Добавить проект" с полями: название, описание, автор (ФИО + TG)
- [ ] Редактирование названия проекта inline
- [ ] Добавление/удаление студентов в проекте
- Notes: Инна: "добавить студентов/редактировать название"

---
**[SS-017][Система] Как система, я хочу не ставить проекты с похожими темами в параллельные залы, чтобы гости не пропускали релевантные проекты.**
- [ ] При генерации расписания: проекты с пересечением тегов > 50% не ставятся в одно время
- [ ] Если невозможно развести все - предупреждение организатору: "Проекты X и Y пересекаются по теме, но стоят параллельно"
- [ ] Организатор может подтвердить или попросить перегенерировать
- Notes: Инна: "6 проектов одновременно LLM не должны быть в параллель"

---

### [EPIC-002b+] Генерация расписания из кластеров (НОВЫЙ)

---
**[US-029][Организатор] Как организатор, я хочу после утверждения кластеризации сгенерировать расписание с временными слотами и перерывами, чтобы не составлять его вручную.**
- [ ] Автогенерация: проекты -> временные слоты по 15 мин
- [ ] Автоматические перерывы (кофе-брейк) каждые N проектов
- [ ] Учет ограничений по времени зала (из US-025)
- [ ] Результат: визуальная сетка "зал x время"
- [ ] Drag-drop для ручной корректировки
- [ ] Кнопки "Кофе-брейк" и "Секция" работают (добавляют блоки в расписание)
- Notes: Инна: "хочется разбиение прям по времени, чтобы можно было внести слоты"

---
**[US-030][Организатор] Как организатор, я хочу выгрузить расписание в разных форматах (XLSX, ICS), чтобы поделиться с модераторами и участниками.**
- [ ] XLSX: полная таблица с ФИО, описаниями, залом, временем (уже реализовано)
- [ ] ICS: календарный файл (уже реализовано)
- [ ] Фильтр по модератору: выгрузка расписания конкретного модератора
- [ ] Фильтр по залу: выгрузка одного зала
- Notes: Инна: "можем выгрузить расписание по модератору его расписание"

### [EPIC-004+] Распределение экспертов (расширение)

---
**[US-031][Организатор] Как организатор, я хочу задать минимальное кол-во экспертов в зале из интерфейса, чтобы система подсвечивала незакрытые залы.**
- [ ] Поле "Мин. экспертов на зал" в настройках матчинга (по умолчанию 2)
- [ ] Если в зале меньше минимума -> статус "Незакрыт" на вкладке покрытия
- Notes: Инна: "количество экспертов в зале минимальное мы можем установить в интерфейсе?"

---
**[SS-018][Система] Как система, я хочу учитывать должность эксперта при матчинге, чтобы распределение было точнее.**
- [ ] Матчинг учитывает: теги интересов + должность (напр. "ML Engineer" -> ML-залы)
- [ ] Нечеткое сопоставление: "Data Scientist" ~ "ML", "Product Manager" ~ "Стартап"
- [ ] Должность = дополнительный сигнал, не замена тегов
- Notes: Инна + созвон: "метчинг должен быть более гибким, например, учитывать должность"

---
**[US-032][Организатор] Как организатор, я хочу видеть временные слоты при распределении экспертов, чтобы назначать их на конкретные промежутки.**
- [ ] В UI матчинга: рядом с залом - доступные временные слоты
- [ ] Можно назначить эксперта на весь зал или на конкретные слоты
- [ ] Визуализация: какие слоты покрыты, какие нет
- Notes: Инна: "хочется чтобы слоты времени отображались, можно было выбирать конкретные слоты"

### [EPIC-018] Экспертный пайплайн в боте (НОВЫЙ)

---
**[US-033][Организатор] Как организатор, я хочу отправить приглашения экспертам через бота с отслеживанием статусов, чтобы видеть кто заинтересован.**
- [ ] Массовая рассылка приглашений через бота
- [ ] Статусы: приглашен -> заинтересован / не заинтересован
- [ ] Дашборд: N приглашено, N заинтересовано, N подтверждено, N отказалось
- Notes: Созвон: "мы пишем и зовем на меро, ставим статусы заинтересован или нет"

---
**[US-034][Эксперт] Как эксперт, я хочу через бота указать свою доступность по дням и слотам, чтобы организаторы учли мое расписание.**
- [ ] Бот: "На какие дни вы доступны?" -> inline-кнопки с датами
- [ ] Бот: "Укажите удобные временные слоты" -> кнопки утро/день/вечер или конкретные часы
- [ ] Вариант "Доступен весь день" -> пропуск выбора слотов
- [ ] Данные передаются в систему матчинга
- Notes: Инна: "сначала ботом собрать время, потом сделать матчинг"

---
**[US-035][Эксперт] Как эксперт без ограничений по времени, я хочу пропустить выбор слотов и сразу получить предложение комнат, чтобы не тратить время на ненужные шаги.**
- [ ] Вариант "Хочу посмотреть все" при выборе доступности
- [ ] Пропуск шага слотов -> сразу матчинг по комнатам
- [ ] Эксперт видит все комнаты с тематиками и выбирает
- Notes: Созвон: "есть эксперты без ограничения по времени, которые хотят посмотреть все"

---
**[SS-019][Система] Как система, я хочу после сбора доступности автоматически предложить эксперту комнату, чтобы ускорить процесс.**
- [ ] Матчинг: теги эксперта + должность + доступные слоты -> подходящая комната
- [ ] Эксперту: "По вашим интересам и доступности предлагаем Зал X (тема: NLP)"
- [ ] Список проектов зала с кратким описанием

---
**[US-036][Эксперт] Как эксперт, я хочу подтвердить или запросить смену предложенной комнаты, чтобы попасть на интересные проекты.**
- [ ] Кнопки: "Подтверждаю" / "Хочу поменять комнату"
- [ ] При подтверждении -> статус "Подтвержден", организатор уведомлен
- [ ] При запросе смены -> организатор получает уведомление в админке
- Notes: Инна: "было бы удобно, чтобы там было две кнопки: подтвердить, хочу поменять комнату"

---
**[SS-020][Система] Как система, я хочу уведомить организатора при запросе эксперта на смену комнаты, чтобы он мог отреагировать.**
- [ ] Уведомление в админке: "Эксперт [имя] хочет поменять комнату [X] на другую"
- [ ] Организатор: принять запрос -> перематчинг / отклонить с комментарием
- [ ] Эксперту: "Ваш запрос принят, новая комната: [Y]" или "Организатор ответил: [комментарий]"
- Notes: Инна: "нам приходит уведомление, что хочет поменять, перезапускаем матчинг"

### [EPIC-020] Чат-поддержка (НОВЫЙ)

---
**[US-039][Любой] Как пользователь бота, я хочу задать вопрос организаторам прямо в боте, чтобы получить ответ без поиска контактов.**
- [ ] Кнопка "Связь с организаторами" в главном меню бота
- [ ] Свободный текст вопроса
- [ ] Бот: "Ваш вопрос передан организаторам. Ответ придет в этот чат"
- [ ] Ответ организатора приходит как сообщение от бота
- Notes: Инна: "могут ли нам задавать вопросы в условном боте, чтобы отображались тут и мы могли отвечать с админки"

---
**[US-040][Организатор] Как организатор, я хочу видеть вопросы пользователей в админке и отвечать на них, чтобы поддержать участников.**
- [ ] Список диалогов в админке: имя, роль, последнее сообщение, время, статус (новый/отвечен)
- [ ] Полная история переписки при клике на диалог
- [ ] Поле ответа + кнопка "Отправить"
- [ ] Фильтр по роли, статусу
- [ ] Счетчик непрочитанных в sidebar
- Notes: До 30-50 одновременных диалогов в день DD

---
**[SS-022][Система] Как система, я хочу уведомить организатора о новом вопросе в чате, чтобы он не пропустил обращение.**
- [ ] Счетчик непрочитанных обновляется каждые 3 сек (polling)
- [ ] Алерт при > 10 непрочитанных
- [ ] Звуковое/визуальное уведомление в админке

### [EPIC-013+] Оценка проектов экспертом (расширение)

---
**[US-041][Эксперт] Как эксперт, я хочу оценить проекты через бота по критериям DD, чтобы результаты сразу попали в систему.**
- [ ] После каждого слота бот предлагает оценить проект
- [ ] 7 критериев (Актуальность, Практ.значимость, Новизна, Импакт, R&D, Масштабирование, Качество реализации)
- [ ] Шкала 1-3 по каждому критерию (как в существующей системе DD)
- [ ] Комментарий в свободной форме (опционально)
- [ ] Пропуск проекта если не смотрел ("Не оценивал")
- [ ] Занимает < 2 мин на проект
- Notes: Расширяет US-019. Критерии из `data/test/experts_22jan_anon.xlsx`

---
**[US-042][Организатор] Как организатор, я хочу видеть результаты оценок экспертов на дашборде, чтобы анализировать итоги DD.**
- [ ] Виджет на дашборде: прогресс оценки (N из M проектов оценено)
- [ ] Средний балл по залам
- [ ] Топ-5 и антитоп-5 проектов
- [ ] Выгрузка в XLSX: все оценки с критериями
- Notes: Инна: "мы потом на дашборде сможем итог видеть и аналитику"

---
**[SS-023][Система] Как система, я хочу отправить общую форму обратной связи после DD, чтобы собрать мнение всех участников.**
- [ ] После DD: всем ролям - "Как прошел Demo Day? Оцените от 1 до 5"
- [ ] Текстовый ответ: "Что бы вы улучшили?"
- [ ] Ответы приходят организатору в админку
- Notes: Инна: "в конце уведомление по общей ОС и чтобы нам текстовый ответ потом пришел"
