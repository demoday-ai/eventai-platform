# User Story Map: AI-агент-куратор DemoDay

> **Версия:** 2.0
> **Дата:** 2026-02-01
> **На основе:** Brief v3.0
> **Изменения v2.0:** 5 ролей (+ Бизнес/партнёр), подтипы гостей для аналитики, актуализация по данным экспертов и форме контрольного рубежа

---

## Backbone (Путь по жизненному циклу Demo Day)

```
[Онбординг] → [Подготовка DD] → [Приглашение и подтверждение] → [Канун DD] → [Во время DD] → [После DD]
```

### Роли

| Роль | Описание | Кол-во |
|------|----------|--------|
| **Организатор** | Core-команда Хаба, составляет расписание, управляет процессом | ~5 чел |
| **Студент** | Автор проекта, выступает на DD | ~330 проектов |
| **Эксперт** | Ментор / жюри, оценивает проекты | ~50 подтв. из 294 |
| **Гость** | Внешний посетитель: абитуриент, AI-практик, другое | ~50 доходимость |
| **Бизнес/партнёр** | Индустриальный партнёр, инвестор, HR/нанимающий менеджер | подмножество гостей |
| **Система** | Автоматические действия бота | — |

> **Примечание:** Гость при входе выбирает подтип (Абитуриент / AI-практик / Другое) — для аналитики организатора. Бизнес/партнёр — отдельная роль с расширенным функционалом (1:1 встречи, LOI, бизнес-фильтры).

### Важные различия

- **Треки** (EdTech, стартап, индустрия, наука) — формат обучения студента
- **Тематики** (NLP, CV, Agents, FinTech, ...) — о чём проект. Кластеризация по тематикам, не по трекам

## Release 1.0 (MVP — демо 6 февраля)

### [EPIC-001] Онбординг и выбор роли
---
**[US-001][Любой] Как пользователь, я хочу начать диалог с ботом и выбрать свою роль, чтобы видеть интерфейс под мои задачи.**
- [ ] /start открывает приветственное сообщение с описанием бота
- [ ] Inline-кнопки: Организатор / Эксперт / Студент / Гость / Бизнес-партнёр
- [ ] После выбора роли бот показывает меню этой роли
- [ ] Можно сменить роль командой /role
- Notes: Единая точка входа для всех — ключевое отличие от Demo-Hero

---
**[US-002][Гость] Как гость, я хочу указать свой подтип (Абитуриент / AI-практик / Другое), чтобы организаторы могли анализировать аудиторию.**
- [ ] После выбора роли "Гость" — второй экран с подтипами
- [ ] Inline-кнопки: Абитуриент / AI-практик / Другое
- [ ] Подтип сохраняется для аналитики, не влияет на функционал бота
- Notes: По данным прошлого DD: ~30% абитуриенты, ~45% AI-практики, ~25% другое

---
**[SS-001][Система] Как система, я хочу запомнить роль, подтип и Telegram ID пользователя, чтобы адресно отправлять сообщения и собирать аналитику.**
- [ ] Telegram ID + роль + подтип (для гостей) сохраняются при первом выборе
- [ ] При повторном /start — предлагает продолжить с текущей ролью или сменить

### [EPIC-002] Загрузка и кластеризация проектов
---
**[US-003][Организатор] Как организатор, я хочу загрузить данные о проектах, чтобы система знала, что кластеризовать.**
- [ ] Организатор отправляет файл (CSV/JSON) или ссылку
- [ ] Бот подтверждает: "Загружено N проектов. Запустить кластеризацию?"
- [ ] Валидация: предупреждение, если у проекта нет описания
- Notes: Для демо — данные предзагружены из `past-demoday-projects.md`. В продакшене: идеал — агент сам забирает из источника; fallback — организатор отправляет файл

---
**[SS-002][Система] Как система, я хочу AI-кластеризовать проекты по тематическим залам, чтобы расписание было тематически когерентным и без дубликатов.**
- [ ] Кластеризация по описаниям проектов + тегам (тематики, не треки)
- [ ] Балансировка: ±5 проектов между залами
- [ ] 0 дубликатов (один проект — один зал)
- [ ] Вывод: список залов с проектами + обоснование кластера
- Notes: Тематики: NLP, CV, Agents, FinTech, EdTech, ML в пром., Security и др. Треки (EdTech, стартап, индустрия, наука) — это формат обучения, не используется для кластеризации

---
**[US-004][Организатор] Как организатор, я хочу просмотреть предложенную кластеризацию и скорректировать её, чтобы утвердить расписание.**
- [ ] Бот показывает кластеры: "Зал 1 (NLP): Проект A, B, C..."
- [ ] Кнопки: Подтвердить / Перенести проект / Перегенерировать
- [ ] После подтверждения — расписание зафиксировано
- Notes: AI предлагает, человек решает

### [EPIC-003] Подтверждение участия студентов
---
**[SS-003][Система] Как система, я хочу разослать каждому студенту его персональный слот, чтобы он знал, когда и где выступает.**
- [ ] Персональное сообщение: "Ты выступаешь [дата], [зал], [время]"
- [ ] Рассылка после утверждения расписания организатором
- Notes: Telegram-контакт студента берётся из формы контрольного рубежа (поле Telegram)

---
**[US-005][Студент] Как студент, я хочу подтвердить или отклонить своё участие, чтобы организаторы знали, кто придёт.**
- [ ] Inline-кнопки: Подтверждаю / Не смогу
- [ ] При "Не смогу" — опционально указать причину
- [ ] Бот подтверждает: "Ок, ты в расписании" или "Передали организатору"

---
**[SS-004][Система] Как система, я хочу эскалировать неподтверждённых студентов организатору, чтобы пустые слоты были выявлены заранее.**
- [ ] Если студент не ответил за 3 дня — повторное сообщение
- [ ] Если не ответил за 5 дней — алерт организатору: "N студентов не подтвердили"
- [ ] Список неподтверждённых с контактами кураторов

### [EPIC-004] Распределение экспертов
---
**[US-006][Организатор] Как организатор, я хочу загрузить список экспертов с тегами интересов, чтобы система предложила распределение по залам.**
- [ ] Загрузка таблицы экспертов (ID, теги, статус DD)
- [ ] Бот подтверждает: "Загружено N экспертов с тегами"
- Notes: Данные имеются — 294 эксперта, 31 тег, 76 подтверждённых (`data/experts-public.json`). Для демо — предзагружены

---
**[SS-005][Система] Как система, я хочу предложить каждому эксперту комнату по его тегам интересов, чтобы матчинг был релевантным.**
- [ ] Матчинг: теги эксперта ↔ тематика зала
- [ ] Приоритет экспертам со статусом "Придёт"
- [ ] Каждому эксперту — персональное приглашение в Telegram
- Notes: Топ-теги: LLM(29), CV(23), NLP(23), EdTech(17), StartUp(16). Слабое покрытие: FinTech(8), Security(2)

---
**[US-007][Эксперт] Как эксперт, я хочу увидеть предложенную комнату, выбрать удобные слоты и подтвердить, чтобы попасть на интересные проекты и совместить с работой.**
- [ ] Сообщение: "Приглашаем на DD! По вашим интересам (NLP, Agents) подходит Зал 3. Вот проекты: ..."
- [ ] Выбор временных слотов: "Какие слоты удобны?" (для совмещения с работой)
- [ ] Кнопки: Иду / Хочу другую комнату / Не смогу
- [ ] При "Хочу другую" — показать альтернативы с тематиками
- Notes: Интервью с экспертом: "Могу не только выбрать комнату, но и тайминг расставить, какие слоты могу, а какие нет"

---
**[US-008][Организатор] Как организатор, я хочу видеть дашборд покрытия экспертами, чтобы знать, какие залы не обеспечены.**
- [ ] По каждому залу: кол-во подтверждённых экспертов / нужно
- [ ] Подсветка: покрыт / частично / нет экспертов
- [ ] Список неответивших экспертов

### [EPIC-005] Профилирование и программа для гостей
---
**[US-009][Гость] Как гость, я хочу рассказать боту о своих интересах, чтобы получить персональную подборку проектов.**
- [ ] Бот предлагает описать интересы свободным текстом
- [ ] AI парсит текст → извлекает тематики, технологии, отрасли
- [ ] Подтверждение: "Я понял, вас интересует: NLP, Agents, EdTech. Верно?"
- [ ] Профилирование занимает ≤2 минуты
- Notes: Гибридный подход — подтип уже выбран на этапе онбординга (US-002), здесь — детализация интересов

---
**[SS-006][Система] Как система, я хочу сформировать персональную программу из 10-15 проектов под профиль гостя, чтобы он не тратил время на случайный выбор.**
- [ ] Подборка: название, описание, зал, время, релевантность
- [ ] Ранжирование по совпадению с профилем
- [ ] Учёт параллельности залов (не ставить два проекта в одно время)
- [ ] Разделение: "обязательно посетить" vs "если останется время"

---
**[US-010][Гость] Как гость, я хочу получить короткие брифы по каждому проекту в подборке, чтобы подготовиться к визиту.**
- [ ] По каждому проекту: 2-3 предложения, теги, автор, зал/время
- [ ] Можно запросить подробнее по конкретному проекту

### [EPIC-013] Оценка проектов экспертом
---
**[US-019][Эксперт] Как эксперт, я хочу быстро оценить проекты прямо в Telegram, чтобы не забыть впечатления.**
- [ ] Бот присылает оценку **после каждого проекта** (не в конце дня — "забывается")

- Notes: Интервью с экспертом: путается в таблице при пропуске студентов. Бот после каждого проекта удобнее, чем всё разом в конце

- **Гостевой follow-up с контактами** — нужен ли обмен контактами между гостями и студентами (не только бизнес)?

- **Бизнес/партнёр и гостевой Q&A — наименее валидированы CustDev.** Митигация: для демо работаем на синтетических данных, валидация после
- **15 эпиков в MVP vs 12 в v1.0.** Добавились EPIC-006 (бизнес-профилирование), EPIC-010 (1:1 встречи), EPIC-015 (бизнес follow-up). Компенсация: EPIC-005/006 и EPIC-014/015 — пары с общей логикой, разница в промптах
