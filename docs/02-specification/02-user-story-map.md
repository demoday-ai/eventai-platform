# User Story Map: AI-агент-куратор DemoDay

> **Версия:** 1.0
> **Дата:** 2026-02-01
> **На основе:** Brief v2.0

---

## Backbone (Путь по жизненному циклу Demo Day)

```
[Онбординг] → [Подготовка DD] → [Приглашение и подтверждение] → [Канун DD] → [Во время DD] → [После DD]
```

### Роли

| Роль | Описание |
|------|----------|
| **Организатор** | Core-команда Хаба (~5 чел), составляет расписание, управляет процессом |
| **Студент** | Автор проекта (~330 проектов), выступает на DD |
| **Эксперт** | Ментор / жюри (~50 чел), оценивает проекты |
| **Гость** | Внешний посетитель (партнёр, инвестор, HR), ищет проекты |
| **Система** | Автоматические действия бота |

## Release 1.0 (MVP — демо 6 февраля)

### [EPIC-001] Онбординг и выбор роли
---
**[US-001][Любой] Как пользователь, я хочу начать диалог с ботом и выбрать свою роль, чтобы видеть интерфейс под мои задачи.**
- [ ] /start открывает приветственное сообщение с описанием бота
- [ ] Inline-кнопки: Организатор / Эксперт / Студент / Гость
- [ ] После выбора роли бот показывает меню этой роли
- [ ] Можно сменить роль командой /role
- Notes: Единая точка входа для всех — ключевое отличие от Demo-Hero

---
**[SS-001][Система] Как система, я хочу запомнить роль и Telegram ID пользователя, чтобы адресно отправлять ему сообщения в рамках его роли.**
- [ ] Telegram ID + роль сохраняются при первом выборе
- [ ] При повторном /start — предлагает продолжить с текущей ролью или сменить

---

### [EPIC-002] Загрузка и кластеризация проектов
---
**[US-002][Организатор] Как организатор, я хочу загрузить данные о проектах, чтобы система знала, что кластеризовать.**
- [ ] Организатор отправляет файл (CSV/JSON) или ссылку на Google Sheet
- [ ] Бот подтверждает: "Загружено N проектов. Запустить кластеризацию?"
- [ ] Валидация: предупреждение, если у проекта нет описания
- Notes: Для демо — загрузка из past-demoday-projects.md

---
**[SS-002][Система] Как система, я хочу AI-кластеризовать проекты по тематическим залам, чтобы расписание было тематически когерентным и без дубликатов.**
- [ ] Кластеризация по описаниям проектов + тегам
- [ ] Балансировка: ±5 проектов между залами
- [ ] 0 дубликатов (один проект — один зал)
- [ ] Вывод: список залов с проектами + обоснование кластера

---
**[US-003][Организатор] Как организатор, я хочу просмотреть предложенную кластеризацию и скорректировать её, чтобы утвердить расписание.**
- [ ] Бот показывает кластеры: "Зал 1 (NLP): Проект A, B, C..."
- [ ] Кнопки: Подтвердить / Перенести проект / Перегенерировать
- [ ] После подтверждения — расписание зафиксировано
- Notes: AI предлагает, человек решает

### [EPIC-003] Подтверждение участия студентов
---
**[SS-003][Система] Как система, я хочу разослать каждому студенту его персональный слот, чтобы он знал, когда и где выступает.**
- [ ] Персональное сообщение: "Ты выступаешь [дата], [зал], [время]"
- [ ] Рассылка после утверждения расписания организатором

---
**[US-004][Студент] Как студент, я хочу подтвердить или отклонить своё участие, чтобы организаторы знали, кто придёт.**
- [ ] Inline-кнопки: ✅ Подтверждаю / ❌ Не смогу
- [ ] При "Не смогу" — опционально указать причину
- [ ] Бот подтверждает: "Ок, ты в расписании" или "Передали организатору"

---
**[SS-004][Система] Как система, я хочу эскалировать неподтверждённых студентов организатору, чтобы пустые слоты были выявлены заранее.**
- [ ] Если студент не ответил за 3 дня — повторное сообщение
- [ ] Если не ответил за 5 дней — алерт организатору: "N студентов не подтвердили"
- [ ] Список неподтверждённых с контактами кураторов

**[US-005][Организатор] Как организатор, я хочу загрузить список экспертов с тегами интересов, чтобы система предложила распределение по залам.**
- [ ] Загрузка таблицы экспертов (имя, Telegram, теги)
- [ ] Бот подтверждает: "Загружено N экспертов с тегами"

---
**[SS-005][Система] Как система, я хочу предложить каждому эксперту комнату по его тегам интересов, чтобы матчинг был релевантным.**

---
**[US-006][Эксперт] Как эксперт, я хочу увидеть предложенную комнату и подтвердить или запросить другую, чтобы попасть на интересные мне проекты.**

**[US-007][Организатор] Как организатор, я хочу видеть дашборд покрытия экспертами, чтобы знать, какие залы не обеспечены.**

### [EPIC-005] Профилирование и программа для гостей
---
**[US-008][Гость] Как гость, я хочу рассказать боту о своих интересах, чтобы получить персональную подборку проектов.**
- [ ] Бот спрашивает: роль (инвестор / HR / партнёр / любопытствующий)
- [ ] Интересы: отрасль, технология, стек, формат партнёрства
- [ ] Можно описать свободным текстом — AI извлечёт профиль
- [ ] Профилирование занимает ≤2 минуты

---
**[SS-006][Система] Как система, я хочу сформировать персональную программу из 10-15 проектов под профиль гостя, чтобы он не тратил время на случайный выбор.**
- [ ] Подборка: название, описание, зал, время, релевантность
- [ ] Ранжирование по совпадению с профилем
- [ ] Учёт параллельности залов (не ставить два проекта в одно время)
- [ ] Разделение: "обязательно посетить" vs "если останется время"

---
**[US-009][Гость] Как гость, я хочу получить короткие брифы по каждому проекту в подборке, чтобы подготовиться к визиту.**
- [ ] По каждому проекту: 2-3 предложения, теги, автор, зал/время
- [ ] Можно запросить подробнее по конкретному проекту

## Gap Analysis & Risks

### Пропущенные функции (для обсуждения)

- **Модератор зала** — отдельная роль? Или это организатор в конкретном зале? Нужен ли ему свой интерфейс?
- **Организация слотов 1:1** (гость ↔ студент) — из ТЗ Ботова, но механика сложная: нужны свободные окна, подтверждение обеих сторон. Отложено на 1.1.
- **Мультиязычность** — на DD есть иностранные гости?
- **Повторное профилирование** — если гость хочет изменить интересы после первой подборки?

### Открытые вопросы

- Формат таблицы экспертов с тегами (ждём от Инны)
- Формат формы контрольного рубежа (ждём от Инны)
- Точное количество залов на следующий DD
- Формат ОС экспертов (свободный текст / структурированная форма)

### Риски

- [⚠️ Risk] **5 дней на 12 эпиков.** Митигация: параллельная разработка (Claude генерирует, команда интегрирует), демо-сценарий с предзагруженными данными
- [⚠️ Risk] **SSH-доступ к VM не получен.** Митигация: fallback на ngrok + локальный запуск
- [⚠️ Risk] **Гостевой Q&A и follow-up — наименее валидированы CustDev.** Митигация: для демо работаем на синтетических данных, валидация после
