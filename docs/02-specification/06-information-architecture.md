---

## 1. Пользователи и их цели

### Основные роли

| Роль | Главная цель | Частота использования |
|---|---|---|
| Организатор (~5) | Покрыть секции экспертами и участниками, видеть подтверждения | Ежедневно в период подготовки |
| Студент (~330) | Подтвердить участие, получить обратную связь от экспертов | Несколько раз до события + после |
| Эксперт (~50) | Быстро найти релевантные проекты и оценить | В дни мероприятия |
| Гость (~50) | Найти 3–5 релевантных проектов, оставить фидбек | В день события |
| Бизнес/партнёр (~15–20) | Найти проекты для инвестиций/найма/партнёрства | В день события + follow-up |

### Подтипы гостей (H16, RICE 66)

| Подтип | Доля | Особенности профилирования |
|---|---|---|
| Абитуриент | ~30% | Интересы по тематикам (NLP, CV...) |
| AI-практик | ~45% | Тематики + стек + стадия |
| Другое | ~25% | Свободный текст |

### Mental Model

- **Организатор:** воспринимает как «панель управления событием» (статусы + покрытие).
- **Студент:** воспринимает как «помощник по участию» (подтверждение + обратная связь).
- **Эксперт/гость/бизнес:** воспринимают как «личный помощник» в Telegram.

### Основная навигация (Telegram Bot)

| Позиция | Элемент | Куда ведёт | Доступ | Приоритет |
|---|---|---|---|---|

| 5 | Оценки/Фидбек | Формы оценок | Эксперт, Гость, Бизнес | Средний |
| 6 | Настройки | Роль/интересы/уведомления | Все | Средний |

### Основная навигация (Admin Console)

| Позиция | Элемент | Куда ведёт | Приоритет |
|---|---|---|---|
| 1 | Дашборд | Обзор по событию | Высокий |
| 2 | Покрытие | Матрица тематик/секций | Высокий |

### Утилитарная навигация

- Поиск: глобальный (проекты, эксперты, секции)
- Уведомления: да (для организатора)
- Помощь: да (FAQ/контакты)

## 4. Ключевые User Flows

### Flow 1: Эксперт подтверждает участие и оценивает проекты

```
Telegram bot → Выбор роли (Эксперт) → Профиль интересов → Подтверждение участия → Напоминание → Оценка проекта → Отправка
```

### Flow 2: Организатор контролирует покрытие

```
Admin Console → Дашборд → Покрытие тематик → Просмотр статусов → Действие (напоминание)
```

### Flow 3: Гость получает рекомендации и оставляет фидбек

```
Telegram bot → Выбор роли (Гость) → Подтип (абитуриент/AI-практик/другое) → Профиль интересов (кнопки → текст) → Рекомендации → Q&A-помощник → Мой список → Фидбек → Хочу контакт
```

### Flow 4: Бизнес/партнёр ищет проекты для партнёрства

```
Telegram bot → Выбор роли (Бизнес/партнёр) → Бизнес-профиль (отрасль + стадия + формат) → Рекомендации → Q&A-помощник → Хочу контакт → Follow-up пакет
```

### Flow 5: Студент подтверждает участие и получает ОС

```
Telegram bot → Выбор роли (Студент) → Подтверждение участия/темы → Напоминание → [после DD] Получение обратной связи
```

## 5. Иерархия контента

### Приоритеты отображения

| Приоритет | Контент | Почему важен |
|---|---|---|
| P1 | Статусы подтверждений / покрытие | Ключевая цель организатора |
| P1 | Рекомендации и избранное | Ключевая цель гостей/бизнеса/экспертов |
| P2 | Q&A-помощник | Повышает качество взаимодействия (гости/бизнес) |
| P2 | Оценки и фидбек | Важны, но после основного действия |
| P3 | Настройки и справка | Вспомогательное |

### Группировка контента

- **Проекты:** по теме/индустрии/стадии, далее по релевантности
- **Секции:** по трекам и статусу подтверждения
- **Оценки:** по проектам и трекам
- **Q&A-подсказки:** по проектам из рекомендаций

## 6. Поиск и обнаружение

### Поиск

- **Тип:** глобальный (в админке) + локальный в рекомендациях
- **Что ищем:** названия проектов, теги, секции, экспертов
- **Фильтры:** тематика, индустрия, стадия, трек, статус подтверждения
- **Сортировка:** по релевантности (по умолчанию), по времени, по стадии

### Обнаружение контента

- Рекомендации: есть (AI-матчинг по профилю интересов)
- Связанные элементы: есть (похожие проекты по тегам)
- Популярное/Недавнее: для организаторов (последние подтверждения)

## 7. Состояния страниц

### Пустые состояния

| Страница | Когда пусто | Что показываем |
|---|---|---|
| Рекомендации | Нет интересов | CTA заполнить профиль |
| Мой список | Нет сохранённых проектов | Подсказка «выбрать из рекомендаций» |
| Покрытие тематик | Нет данных по секциям | Напоминание заполнить секции |
| Q&A-помощник | Нет выбранного проекта | Подсказка «выберите проект из рекомендаций» |

### Состояния загрузки

- Первая загрузка: скелетон (список проектов/матрица)
- Подгрузка данных: спиннер

### Ошибки

- Нет сети: повторить запрос
- Нет доступа: сообщение о роли
- 404: ссылка устарела
