## 1. Пользователи и их цели

### Основные роли

| Роль | Главная цель | Частота использования |
|------|--------------|----------------------|
| Организатор (3-5 чел) | Подготовить Demo Day от загрузки данных до отправки приглашений | Ежедневно в период подготовки (2-4 недели до DD) |

### Mental Model

Организатор воспринимает систему как **пошаговый конвейер**:
- «Загрузил данные → система распределила → я проверил и одобрил → система разослала»
- Каждый этап — чекпоинт, который нужно пройти
- Между чекпоинтами можно вернуться и поправить
- Параллельно можно делать рассылки и смотреть аудиторию

## 2. Appmap (Карта страниц)

```
EventAI Admin
│
├── Login
│
├── Dashboard
│   ├── Global Stepper (Данные → Распределение → Запуск)
│   ├── Дата события + обратный отсчёт
│   ├── Метрики (проекты, студенты, эксперты, партнёры, залы)
│   ├── Quick Action (следующий незавершённый шаг)
│   └── Покрытие залов экспертами (сводка)
│
├── [Подготовка]
│   │
│   ├── Импорт
│   │   ├── Таб: Событие (создание / редактирование)
│   │   ├── Таб: Проекты (загрузка XLSX)
│   │   ├── Таб: Студенты (загрузка XLSX, привязка к проектам)
│   │   ├── Таб: Эксперты (загрузка XLSX)
│   │   └── Таб: Партнёры (загрузка XLSX)
│   │
│   └── Проекты (read-only каталог)
│       └── Фильтры: зал, теги, поиск
│
├── [Распределение]
│   │
│   ├── Кластеризация
│   │   ├── Шаг 1: Параметры (кол-во залов, критерии)
│   │   ├── Шаг 2: Результат (визуализация залов + проекты)
│   │   ├── Шаг 3: Ручные перемещения (drag-and-drop)
│   │   └── Шаг 4: Одобрение
│   │
│   ├── Эксперты
│   │   ├── Таб: Матчинг (запуск → результат → рекомендации → одобрение)
│   │   ├── Таб: Список (CRUD)
│   │   └── Таб: Покрытие (сводка по залам + пробелы)
│   │
│   └── Расписание
│       ├── Шаг 1: Генерация
│       ├── Шаг 2: Просмотр (залы × слоты)
│       └── Шаг 3: Одобрение
│
├── [Коммуникация]
│   │
│   ├── Рассылки
│   │   ├── Новая рассылка (аудитория → фильтры → шаблон → предпросмотр → отправка)
│   │   └── История рассылок
│   │
│   └── Напоминания
│       ├── Автоматические (настройка eve-of-DD, pre-slot)
│       ├── Брифинг экспертам (предпросмотр → массовая отправка)
│       └── Подтверждение участия (отправка → сводка → повтор неответившим)
│
├── [Аналитика]
│   │
│   └── Аудитория
│       ├── Список контактов (таблица + фильтры + поиск)
│       ├── Карточка контакта (профиль + история взаимодействий)
│       └── Сегменты (фильтрация → сохранение → рассылка)
│
└── [Администрирование]
    │
    ├── Настройки
    │   └── Управление организаторами (добавить / удалить по Telegram)
    │
    └── Журнал действий
        └── Таблица: дата, пользователь, действие, детали
```

### Описание разделов

| Раздел | Назначение | Доступ | Доступен всегда? |
|--------|-----------|--------|------------------|
| Dashboard | Прогресс, метрики, следующий шаг | Организаторы | Да (empty state без события) |
| Импорт | Создание события, загрузка всех данных | Организаторы | Да |
| Проекты | Просмотр каталога проектов | Организаторы | Да (empty state без проектов) |
| Кластеризация | AI-группировка проектов по залам | Организаторы | Да (empty state без проектов) |
| Эксперты | Матчинг, управление, покрытие | Организаторы | Да (empty state без кластеризации для матчинга) |
| Расписание | Генерация и утверждение сетки | Организаторы | Да (empty state без кластеризации) |
| Рассылки | Ручная отправка сообщений | Организаторы | Да (empty state без участников) |
| Напоминания | Автоматические уведомления и брифинг | Организаторы | Да (empty state без расписания) |
| Аудитория | Контакты из бота, сегментация | Организаторы | Да (empty state без контактов) |
| Настройки | Управление доступом | Организаторы | Да |
| Журнал | Аудит действий | Организаторы | Да |

### Sidebar (основная навигация)

| Группа | Пункт | Статус-бейдж | Иконка (Lucide) |
|--------|-------|--------------|-----------------|
| — | Dashboard | — | `LayoutDashboard` |
| Подготовка | Импорт | Серый / Зелёный | `Upload` |
| Подготовка | Проекты | Серый / Зелёный | `FolderOpen` |
| Распределение | Кластеризация | Серый / Жёлтый / Зелёный | `Grid3x3` |
| Распределение | Эксперты | Серый / Жёлтый / Зелёный | `Users` |
| Распределение | Расписание | Серый / Жёлтый / Зелёный | `Calendar` |
| Коммуникация | Рассылки | — | `Send` |
| Коммуникация | Напоминания | Серый / Зелёный | `Bell` |
| Аналитика | Аудитория | — | `Eye` |
| Администрирование | Настройки | — | `Settings` |
| Администрирование | Журнал | — | `ScrollText` |

**Логика статус-бейджей:**
- Серый (круг) — не начато
- Жёлтый (круг) — требует внимания / в процессе
- Зелёный (галочка) — завершено
- Без бейджа — инструмент без конечного состояния (Рассылки, Аудитория)

### Global Stepper (верхняя панель)

Горизонтальная полоса прогресса с 3 фазами:

```
● Данные ──── ○ Распределение ──── ○ Запуск
```

- **Данные:** завершена, когда загружены проекты + студенты + эксперты + событие создано
- **Распределение:** завершена, когда кластеризация + матчинг + расписание одобрены
- **Запуск:** завершена, когда напоминания настроены + брифинг отправлен

Клик на фазу → переход на первую незавершённую страницу этой фазы.

### Утилитарная навигация

| Элемент | Расположение | Описание |
|---------|-------------|----------|
| Имя организатора | Верхний правый угол | Telegram-username + кнопка выхода |
| Глобальный поиск | Нет | Не нужен — навигация по sidebar достаточна при 11 страницах |
| Уведомления | Нет (v1.0) | Auto-refresh заменяет push-уведомления |
| Помощь | Нет (v1.0) | Будет в онбординг-туре (Release 1.1) |

## 4. Ключевые User Flows

### Flow 1: Первая настройка (Happy Path)

```
Dashboard (empty state)
    │ клик «Создайте событие»
    ▼
Импорт → Таб «Событие»
    │ заполнил название + даты → Сохранить
    ▼
Импорт → Таб «Проекты»
    │ скачал шаблон → заполнил → загрузил → предпросмотр → подтвердил
    ▼
Импорт → Таб «Студенты»
    │ скачал шаблон → заполнил → загрузил → валидация привязок → исправил inline → подтвердил
    ▼
Импорт → Таб «Эксперты»
    │ скачал шаблон → заполнил → загрузил → подтвердил
    ▼
Импорт → Таб «Партнёры»
    │ загрузил → подтвердил
    ▼
Dashboard (данные загружены, Quick Action → «Запустить кластеризацию»)
```

### Flow 2: Распределение

```
Dashboard (Quick Action: «Запустить кластеризацию»)
    │
    ▼
Кластеризация → Параметры → Запуск
    │ AI обработал
    ▼
Кластеризация → Результат
    │ проверил, переместил проекты
    ▼
Кластеризация → Одобрить
    │ подсказка: «Далее — эксперты»
    ▼
Эксперты → Таб «Матчинг» → Запуск
    │ AI распределил
    ▼
Эксперты → Таб «Покрытие»
    │ проверил пробелы
    │ [если пробелы] → рекомендации → назначил
    ▼
Эксперты → Таб «Матчинг» → Одобрить
    │ подсказка: «Далее — расписание»
    ▼
Расписание → Генерация → Просмотр → Одобрить
    │
    ▼
Dashboard (все фазы зелёные)
```

### Flow 3: Рассылка (доступен в любой момент)

```
Рассылки
    │
    ▼
Выбор аудитории (студенты / эксперты / партнёры / все)
    │
    ▼
Фильтры (по залу, проекту, тегам)
    │
    ▼
Шаблон сообщения (переменные: {имя}, {зал}, {проект})
    │
    ▼
Предпросмотр
    │
    ▼
«Отправить 47 человекам?» → Подтвердить
    │
    ▼
История рассылок (новая запись)
```

### Flow 4: Запуск коммуникаций

```
Напоминания → Настройка автоматических
    │ выбрал: eve-of-DD + pre-slot → Сохранить
    ▼
Напоминания → Брифинг экспертам
    │ предпросмотр → Отправить всем
    ▼
Напоминания → Подтверждение участия
    │ отправил запросы → ждёт
    ▼
Напоминания → Сводка подтверждений
    │ [неответившие] → Повторить отправку
```

### Flow 5: Мониторинг аудитории

```
Аудитория → Список контактов
    │ фильтр: роль = «гость», активность > 3
    ▼
Аудитория → Карточка контакта
    │ профиль, интересы, какие проекты смотрел
    ▼
Аудитория → Создать сегмент → Сохранить
    │
    ▼
«Отправить рассылку этому сегменту» → Рассылки (с предзаполненной аудиторией)
```

## 5. Иерархия контента

### Приоритеты отображения

| Приоритет | Контент | Почему важен |
|-----------|---------|-------------|
| P1 (Критический) | Global Stepper + Quick Action | Определяет, что делать дальше — без этого пользователь потерян |
| P1 (Критический) | Метрики (проекты, эксперты, залы) | Даёт мгновенное понимание масштаба и готовности |
| P1 (Критический) | Покрытие залов экспертами | Главная боль организатора — пробелы видны сразу |
| P2 (Важный) | Дата + обратный отсчёт | Контекст срочности |
| P2 (Важный) | Статус-бейджи в sidebar | Прогресс без перехода на страницу |
| P2 (Важный) | Empty states с подсказками | Навигация при отсутствии данных |
| P3 (Вспомогательный) | История рассылок | Нужна для аудита, но не для основной задачи |
| P3 (Вспомогательный) | Журнал действий | Аудит, редко используется |
| P3 (Вспомогательный) | Аналитика аудитории | Становится важной ближе к DD и после |

### Группировка контента внутри разделов

| Раздел | Организация | Сортировка по умолчанию |
|--------|------------|------------------------|
| Проекты | Таблица с фильтрами | По названию |
| Эксперты → Список | Таблица с поиском | По имени |
| Аудитория | Таблица с фильтрами | По дате последнего контакта (новые сверху) |
| Рассылки → История | Таблица | По дате (новые сверху) |
| Журнал | Таблица | По дате (новые сверху) |
| Покрытие | Карточки по залам | По номеру зала |

## 7. Состояния страниц

### Пустые состояния

| Страница | Условие | Что показываем |
|----------|---------|----------------|
| Dashboard | Нет события | «Нет активного мероприятия. Создайте событие на странице Импорта» + кнопка → Импорт |
| Dashboard | Есть событие, нет данных | Метрики = 0, Quick Action → «Загрузите проекты» |
| Проекты | Нет проектов | «Проекты ещё не загружены» + кнопка → Импорт/Проекты |
| Кластеризация | Нет проектов | «Для кластеризации загрузите проекты» + кнопка → Импорт/Проекты |
| Эксперты → Матчинг | Кластеризация не одобрена | «Для матчинга одобрите кластеризацию» + кнопка → Кластеризация |
| Эксперты → Список | Нет экспертов | «Загрузите экспертов» + кнопка → Импорт/Эксперты |
| Расписание | Кластеризация/матчинг не одобрены | «Для расписания одобрите кластеризацию и матчинг» + ссылки |
| Рассылки | Нет события | «Создайте мероприятие на странице Импорта» + кнопка → Импорт |
| Рассылки | Нет участников | «Загрузите участников на странице Импорта» + кнопка → Импорт |
| Рассылки | Есть участники, нет истории | «Рассылок пока не было. Отправьте первую» + форма |
| Напоминания | Расписание не одобрено | «Для напоминаний утвердите расписание» + кнопка → Расписание |
| Аудитория | Нет события | «Создайте мероприятие на странице Импорта» + кнопка → Импорт |
| Аудитория | Нет контактов | «Контакты появятся, когда участники начнут использовать бота» |
| Импорт → Студенты | Нет проектов | «Для привязки студентов сначала загрузите проекты» + ссылка на таб Проекты |

**Принцип:** ни одна страница не заблокирована. Все доступны для просмотра. Empty state объясняет ситуацию и даёт ссылку на нужный шаг.

### Состояния загрузки

| Ситуация | Отображение |
|----------|-------------|
| Загрузка страницы | Скелетон (skeleton) — контуры блоков |
| AI-обработка (кластеризация, матчинг) | Прогресс-бар + текст: «Кластеризация... это может занять до минуты» |
| Загрузка файла | Прогресс-бар загрузки |
| Auto-refresh данных | Невидимый, без индикации (данные обновляются фоново) |

### Ошибки

| Ситуация | Отображение |
|----------|-------------|
| Ошибка загрузки файла | Inline-сообщение под drag-area: «Формат не поддерживается» / «Файл повреждён» |
| Ошибки валидации импорта | Таблица ошибок поимённо + кнопка «Скачать отчёт» |
| Ошибка API | Toast-уведомление: «Не удалось сохранить. Попробуйте ещё раз» + кнопка «Повторить» |
| Нет сети | Баннер вверху: «Нет подключения к серверу. Данные могут быть неактуальны» |
| 404 (несуществующая страница) | Страница: «Страница не найдена» + ссылка на Dashboard |
| 403 (нет доступа) | Страница: «Доступ запрещён. Обратитесь к администратору» |

## 9. Вторичная навигация (табы и wizard-шаги)

### Страницы с табами

| Страница | Табы | Логика |
|----------|------|--------|
| Импорт | Событие, Проекты, Студенты, Эксперты, Партнёры | Все табы всегда видны. «Событие» — первый таб. Порядок = рекомендуемая последовательность загрузки |
| Эксперты | Матчинг, Список, Покрытие | Все табы всегда доступны |

### Страницы с wizard-шагами

| Страница | Шаги | Логика |
|----------|------|--------|

### B. Принципы визуала (из USM)

1. Минимализм — каждый элемент несёт смысл
2. Иконки — только Lucide React (SVG), никаких unicode-эмоджи
3. Типографика как навигация — 3 уровня максимум
4. Цвет функциональный — только статусы (зелёный/жёлтый/серый) + один акцентный
5. Воздух важнее контента — лучше скролл, чем перегруженный экран
6. Одно действие на экран — одна primary-кнопка
