# UX-спецификация: Админ-панель EventAI

## Философия

Организатор Demo Day -- не разработчик. Панель должна вести за руку через процесс подготовки события, но не блокировать свободную навигацию для команды из 2-3 человек, которые работают параллельно.

**Правило:** никаких unicode-эмоджи в UI. Только Lucide React иконки (SVG).

---

## 1. Global Stepper (header)

Горизонтальный прогресс-бар с **3 крупными фазами**, каждая раскрывается на детальные подшаги.

### Фаза 1: Данные

Подшаги: Событие -> Проекты -> Эксперты.

- Статус: зелёный когда загружены проекты + эксперты + теги
- Клик -> переход на `/import`

### Фаза 2: Распределение

Подшаги: Кластеризация -> Матчинг -> Расписание.

- Статус: зелёный когда кластеризация одобрена + матчинг одобрен + расписание одобрено
- Клик -> переход на `/clustering`

### Фаза 3: Запуск

Подшаги: Приглашения -> Напоминания.

- Статус: зелёный когда приглашения отправлены
- Клик -> переход на `/messaging`

### Визуально

```
[*  Данные] -------- [o  Распределение] -------- [o  Запуск]
```

- `*` заполненный (Check icon) = фаза завершена
- половина (Loader icon) = в процессе
- `o` пустой (Circle icon) = ещё не начата

При наведении/клике на фазу -- dropdown показывает подшаги с их статусами (Check / Circle).

### Логика определения статуса (из API)

- **Данные:** `projects_count > 0` AND `experts_count > 0`
- **Распределение:** `clustering.approved_at != null` AND `matching.approved_at != null` AND `schedule.approved_at != null`
- **Запуск:** `invites_sent > 0`

### Иконки (Lucide)

Stepper: `Check`, `Circle`, `Loader2`. Статусы: `CheckCircle`, `Clock`, `XCircle`, `AlertCircle`.

## 2. Dashboard (`/` -- landing page)

### Состояние "нет события"

- Баннер: "Нет активного мероприятия"
- Карточки-инструкции: "Загрузите проекты -> Запустите кластеризацию -> Отправьте приглашения"
- Кнопка "Начать подготовку" -> `/import`

### Состояние "есть данные"

- Метрики: Проекты | Эксперты | Гости | Залы (как сейчас)
- Алерты (как сейчас)
- **Покрытие залов** -- таблица + кнопка "Детали" -> `/coverage/rooms/:id`
- **Quick Actions** -- карточки-ссылки на следующий незавершённый шаг:
  - "Кластеризация не одобрена -> Перейти"
  - "Расписание не сгенерировано -> Перейти"
  - "Приглашения не отправлены -> Перейти"

## 5. Страницы подробно

### `/import` -- Импорт данных

Табы: **Событие | Проекты | Эксперты | Гости**

- **Таб "Событие":** создание/редактирование события (название, даты) + управление тегами (добавление, LLM-подсказки). Перенос из текущего Settings.
- **Таб "Проекты":** upload CSV/XLSX/JSON (как сейчас DataImport, секция проектов)
- **Таб "Эксперты":** upload экспертов (как сейчас DataImport, секция экспертов)
- **Таб "Гости":** upload гостей с выбором подтипа (как сейчас DataImport, секция гостей)

### `/projects` -- Список проектов

Без изменений. Read-only список с фильтрами (зал, статус, поиск). Auto-refresh 60s.

### `/clustering` -- Кластеризация

Без изменений. Wizard 4 шага: Параметры -> Результат -> Перемещение -> Одобрение.

### `/experts` -- Эксперты

Табы: **Матчинг | Список экспертов | Покрытие**

- **Таб "Матчинг":** wizard (как сейчас ExpertMatching -- 3 шага)
- **Таб "Список":** CRUD экспертов (как сейчас ExpertList)
- **Таб "Покрытие":** Coverage summary + пробелы + эскалации (как сейчас Coverage, 3 таба внутри)

### `/schedule` -- Расписание

Без изменений. Wizard 3 шага: Генерация -> Просмотр -> Одобрение.

### `/messaging` -- Рассылки

Из текущего Messaging, табы "Обзор" + "Рассылка":

- Секция "Отправить рассылку": выбор аудитории, шаблон, предпросмотр, отправка
- Секция "История": таблица отправленных рассылок + статистика

### `/reminders` -- Напоминания

Из текущего Messaging, табы "Напоминания" + "Участие" + "Брифинг":

- Напоминания по расписанию (eve-of-DD, pre-slot)
- Подтверждение участия (рассылка + сводка + неподтверждённые)
- Брифинг экспертам (предпросмотр + отправка)

### `/guests` -- Гости и партнёры

Без изменений. Справочник: таблица с поиском, фильтрами, expand-детали.

### `/settings` -- Настройки

Только: управление организаторами (CRUD таблица + добавление по Telegram username).

### `/audit` -- Журнал действий

Только: таблица audit log с фильтрами по типу действия. Перенос из текущего Settings.

### `/rooms/:id` -- Детали зала

Без изменений. Редактирование названия/тематики, список экспертов и проектов.

### `/coverage/rooms/:id` -- Покрытие зала

Без изменений. Детальное покрытие конкретного зала.

## 6. Empty states

Каждая страница, зависящая от события, при отсутствии данных показывает понятный empty state со ссылкой на нужный шаг:

| Страница | Без события | Без данных |
|----------|------------|-----------|
| Dashboard | "Нет активного мероприятия" + кнопка "Начать" | Метрики = 0, Quick Actions |
| Проекты | "Создайте событие на вкладке Импорт" -> `/import` | "Проекты не загружены" -> `/import` |
| Кластеризация | "Сначала загрузите проекты" -> `/import` | "Запустите кластеризацию" (кнопка на месте) |
| Эксперты | "Сначала загрузите экспертов" -> `/import` | "Запустите матчинг" (кнопка на месте) |
| Расписание | "Сначала одобрите кластеризацию" -> `/clustering` | "Сгенерируйте расписание" (кнопка на месте) |
| Рассылки | "Сначала одобрите матчинг" -> `/experts` | -- |
| Гости | "Нет активного мероприятия" -> `/import` | "Гостей пока нет" |

Каждый empty state содержит **кнопку-ссылку** на нужный шаг (не просто текст).

```

/audit                -> Журнал действий

/coverage/rooms/:id   -> Покрытие зала (детали)
```
