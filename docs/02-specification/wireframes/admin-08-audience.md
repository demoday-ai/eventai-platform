# Wireframe: Аудитория

**Контекст:** Организатор просматривает контакты из бота, анализирует интересы, создаёт сегменты
**Цель пользователя:** Понять, кто взаимодействует с ботом, и отправить релевантные сообщения

## Аннотации

| # | Элемент | Поведение |
|---|---------|-----------|
| 1 | Поиск | Мгновенная фильтрация таблицы по имени или @telegram |
| 2 | Фильтры | Dropdown. Роль: Гость/Партнёр/Все. Активность: >N взаимодействий |
| 3 | Чекбоксы в таблице | Множественный выбор для сегментации и рассылок |
| 4 | «Сохранить как сегмент» | Сохраняет текущие фильтры + выбор как именованный сегмент |
| 5 | «Отправить рассылку» | Переход на страницу Рассылки с предзаполненным списком получателей |
| 6 | Строка в таблице (клик) | Переход на карточку контакта |
| 7 | Карточка: интересы | Теги, извлечённые из диалогов бота |
| 8 | Карточка: проекты | Проекты, которые контакт смотрел. ℹ — количество вопросов к проекту |

## Состояния

- **Empty state:** «Пока никто не взаимодействовал с ботом. Контакты появятся автоматически, когда участники начнут использовать бота»

## Переходы

- Строка таблицы → Карточка контакта
- ← Назад → Список контактов
- «Отправить рассылку» → Рассылки (с предзаполненной аудиторией)
