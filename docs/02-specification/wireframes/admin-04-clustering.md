# Wireframe: Кластеризация

**Контекст:** Организатор группирует проекты по залам с помощью AI
**Цель пользователя:** Получить разумное распределение проектов по залам, при необходимости скорректировать

## Аннотации

| # | Элемент | Поведение |
|---|---------|-----------|
| 1 | Wizard-степпер | ○ = не пройден, ● = пройден. Клик на пройденный шаг → возврат |
| 2 | Карточки залов | Сетка 2 колонки. Название зала + тематика + кол-во проектов + список |
| 3 | Drag-and-drop | Перетаскивание проекта из одного зала в другой. Альтернатива: ⋮ → «Переместить в...» |
| 4 | «Перезапустить» | Secondary button. Возврат к параметрам для повторной кластеризации |
| 5 | «Одобрить» | Primary button. Диалог подтверждения: «Зафиксировать распределение?» |
| 6 | Подсказка после одобрения | «Кластеризация одобрена. Следующий шаг — распределение экспертов» + ссылка |

## Переходы

- Одобрить → обновление Global Stepper + подсказка → Эксперты/Матчинг
- Перезапустить → шаг «Параметры»
- Карточка зала (клик на «...ещё N») → раскрытие полного списка
