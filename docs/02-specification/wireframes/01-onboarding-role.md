# Wireframe: Онбординг — выбор роли

> **Платформа:** Mobile (Telegram bot, 375px)
> **Контекст:** Первый вход пользователя в бот
> **Цель пользователя:** Выбрать роль и начать сценарий
> **Версия:** 1.1 (обновлено: 5 ролей + подтип гостя)

## Аннотации

| # | Элемент | Поведение |
|---|---------|-----------|
| 1 | Выбор роли (5 вариантов) | Сохраняется в user_roles. Бизнес/партнёр — отдельная роль с расширенным профилированием |
| 2 | Код приглашения | Для организаторов по коду; опционально. Показывается только при выборе «Организатор» |
| 3 | Запросить доступ | Отправляет access_request; показывается только при выборе «Организатор» |
| 4 | CTA «Продолжить» | Активируется после выбора роли (и валидного кода, если организатор) |
| 5 | Подтип гостя (шаг 1.5) | Показывается только для роли «Гость». Сохраняется в users.guest_subtype. Влияет на ранжирование рекомендаций |

## Переходы

- Организатор → Шаг 2: «Профиль интересов» (пропускается) → Admin Console
- Студент → Шаг 2: «Подтверждение участия»
- Эксперт → Шаг 2: «Профиль интересов» (кнопки тематик)
- Гость → Шаг 1.5: «Подтип гостя» → Шаг 2: «Профиль интересов» (кнопки → текст)
- Бизнес/партнёр → Шаг 2: «Бизнес-профиль» (отрасль + формат + стадия) → «Профиль интересов»
