# Wireframe: Расписание

**Контекст:** Организатор генерирует и утверждает расписание выступлений
**Цель пользователя:** Получить корректную сетку залы x слоты и утвердить

## Аннотации

| # | Элемент | Поведение |
|---|---------|-----------|
| 1 | Wizard-степпер | Линейный: Генерация → Просмотр → Одобрение |
| 2 | Сетка | Горизонтальный скролл для >4 залов. Каждая ячейка: название проекта + тимлид |
| 3 | Ячейка слота | При наведении: полное название + теги. Слот = 15 мин |
| 4 | «Перегенерировать» | Secondary. Возврат к параметрам генерации |
| 5 | «Одобрить» | Primary. Диалог: «Утвердить расписание? После утверждения можно отправлять напоминания» |

## Состояния

- **Empty state:** «Для генерации расписания одобрите кластеризацию и матчинг» + ссылки
- **Loading:** Прогресс-бар: «Генерация расписания...»

## Переходы

- Одобрить → Global Stepper: фаза «Распределение» завершена. Подсказка: «Расписание утверждено. Можно отправить напоминания» + ссылка → Напоминания
