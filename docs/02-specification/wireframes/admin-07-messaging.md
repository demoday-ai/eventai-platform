# Wireframe: Рассылки

**Контекст:** Организатор отправляет сообщения участникам на любом этапе подготовки
**Цель пользователя:** Быстро составить и отправить релевантное сообщение нужной аудитории

## Аннотации

| # | Элемент | Поведение |
|---|---------|-----------|
| 1 | Радио-кнопки аудитории | Одиночный выбор. При выборе — обновляется счётчик «Получателей» |
| 2 | Фильтры (зал, теги) | Dropdown. Фильтрация внутри выбранной аудитории |
| 3 | Счётчик получателей | Обновляется мгновенно при изменении аудитории/фильтров |
| 4 | Переменные | Подсказка над textarea. Клик на переменную → вставка в текст |
| 5 | Предпросмотр | Показывает первого получателя с подставленными переменными |
| 6 | «Отправить N чел.» | Primary. Диалог подтверждения: «Отправить сообщение 412 участникам?» |

## Состояния

- **Empty state (нет участников):** «Загрузите участников на странице Импорта» + ссылка
- **Empty state (история пуста):** «Рассылок пока не было. Отправьте первую» + ссылка на таб «Новая рассылка»

## Переходы

- Отправить → переход на таб «История», новая запись сверху
- Аудитория → Рассылки (из страницы Аудитория, с предзаполненным сегментом)
