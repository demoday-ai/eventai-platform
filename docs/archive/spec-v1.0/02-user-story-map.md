# User Story Map: AI-first Unconference Navigator

## Backbone (Путь пользователя)

Присоединение/роль → Профиль интересов → Поиск и сопоставление → Подтверждение участия → Взаимодействие на событии → Обратная связь → Фоллоу‑ап

---

## Release 1.0 (MVP)

### [EPIC-001] Роли и контекст
---
**[US-001][Гость] Как гость, я хочу выбрать роль, чтобы получать релевантные рекомендации и сценарии.**
- [ ] Можно выбрать роль при первом входе
- [ ] Роль сохраняется для дальнейших действий
- [ ] Переход на следующий шаг доступен после выбора роли
---
**[US-002][Эксперт/Участник/Организатор] Как пользователь, я хочу выбрать свою роль, чтобы видеть релевантные задачи и подтверждения.**
- [ ] Роль определяет доступные действия (подтверждения, заявки, обзор покрытия)
- [ ] Роль можно изменить до начала активной работы
- [ ] Роль «Организатор» доступна только по приглашению или назначению админом
---
**[SS-001][Система] Как система, я хочу применять сценарии по роли, чтобы показывать релевантные функции.**
- [ ] Для каждой роли доступны только её основные сценарии
- [ ] Роль влияет на ранжирование и подсказки
- [ ] Система проверяет приглашение или назначение для роли «Организатор»

### [EPIC-002] Профиль интересов
---
**[US-003][Все роли] Как пользователь, я хочу выбрать темы, индустрии и стадии проектов, чтобы получать релевантные рекомендации.**
- [ ] Можно выбрать несколько тем/индустрий/стадий
- [ ] Выбор можно изменить
---
**[US-004][Все роли] Как пользователь, я хочу добавить свободные интересы, чтобы уточнить релевантность.**
- [ ] Свободный ввод добавляется к профилю интересов
- [ ] Система учитывает свободные интересы в рекомендациях
---
**[US-004a][Все роли] Как пользователь, я хочу быстро уточнить профиль интересов после первых рекомендаций, чтобы улучшить подбор.**
- [ ] Можно скорректировать выбранные темы/индустрии/стадии после просмотра рекомендаций
- [ ] Изменения сразу влияют на список рекомендаций
---
**[SS-002][Система] Как система, я хочу формировать профиль интересов, чтобы использовать его в сопоставлении.**
- [ ] Профиль интересов состоит из выбранных и введённых данных
- [ ] Профиль доступен для дальнейших сценариев

### [EPIC-003] Поиск и сопоставление проектов/секций
---
**[US-005][Гость] Как гость, я хочу быстро найти 3–5 релевантных проектов, чтобы подготовить вопросы.**
- [ ] Есть список проектов, релевантных профилю интересов
- [ ] Можно сохранить 3–5 проектов в список интереса
---
**[US-006][Эксперт] Как эксперт, я хочу находить релевантные проекты, чтобы выбрать, где участвовать.**
- [ ] Есть список проектов по моему профилю интересов
- [ ] Можно отметить интерес к проекту
---
**[US-007][Организатор] Как организатор, я хочу фильтровать проекты по тематике, индустрии и стадии, чтобы оценивать покрытие.**
- [ ] Фильтры по тематике, индустрии и стадии доступны
- [ ] Фильтрация влияет на список и статистику покрытия
---
**[SS-003][Система] Как система, я хочу рекомендовать релевантные проекты, чтобы ускорить подбор.**
- [ ] Рекомендации учитывают профиль интересов
- [ ] Рекомендации можно пересматривать и уточнять

### [EPIC-004] Подтверждение участия
---
**[US-008][Участник] Как участник, я хочу подать заявку в секцию и указать тему, дату и время, чтобы подтвердить участие.**
- [ ] Можно подать заявку в секцию
- [ ] В заявке указываются тема, дата и время
- [ ] Статус заявки отображается
---
**[US-009][Участник] Как участник, я хочу подтвердить тему, дату и время, чтобы зафиксировать выступление.**
- [ ] Подтверждение фиксирует выбранные параметры
- [ ] После подтверждения параметры отображаются как согласованные
---
**[US-010][Эксперт] Как эксперт, я хочу подтвердить участие и указать доступные даты/время и тему, чтобы быть учтённым в программе.**
- [ ] Эксперт может подтвердить участие
- [ ] Даты/время и тема сохраняются
---
**[SS-004][Система] Как система, я хочу вести статусы заявок, чтобы организаторы видели подтверждения.**
- [ ] Есть статусы «подано» и «подтверждено»
- [ ] Статус отображается организаторам

### [EPIC-005] Дедлайны и напоминания
---
**[SS-005][Система] Как система, я хочу напоминать о дедлайнах и подтверждениях, чтобы снизить просрочки.**
- [ ] Напоминания отправляются экспертам и участникам
- [ ] Напоминания связаны с ключевыми сроками

### [EPIC-006] Организаторское покрытие тематик
---
**[US-011][Организатор] Как организатор, я хочу видеть покрытие тематик и секций экспертами и участниками, чтобы управлять программой.**
- [ ] Видна матрица/сводка по тематикам и секциям
- [ ] Видны статусы подтверждений
---
**[US-012][Организатор] Как организатор, я хочу видеть непокрытые тематики, чтобы вовремя найти экспертов.**
- [ ] Есть список тематик без подтверждений
- [ ] Для каждой темы видны недостающие роли

### [EPIC-007] Взаимодействие на событии
---
**[US-013][Гость] Как гость, я хочу иметь список выбранных проектов, чтобы быстро вернуться к ним во время Demo Day.**
- [ ] Список выбранных проектов доступен во время события
- [ ] Можно открыть карточку проекта из списка
---
**[SS-006][Система] Как система, я хочу показывать ключевую информацию о проекте, чтобы помочь подготовить вопросы.**
- [ ] Карточка проекта содержит тему, трек и стадию
- [ ] Карточка доступна из рекомендаций и списка интереса

### [EPIC-008] Обратная связь
---
**[US-014][Эксперт] Как эксперт, я хочу оценить проект и оставить комментарий, чтобы зафиксировать мнение.**
- [ ] Оценка по шкале 1–5
- [ ] Комментарий обязателен при оценке
- [ ] Можно оценить глубину проработки и качество выступления
---
**[US-015][Эксперт] Как эксперт, я хочу оценивать проект по трековым критериям, чтобы учитывать специфику.**
- [ ] Для стартапов есть критерий «бизнес‑составляющая»
- [ ] Для научного трека есть критерий «научная глубина»
- [ ] Для индустрии есть критерий «внедрение и импакт»
---
**[US-015a][Эксперт] Как эксперт, я хочу видеть краткие подсказки по трековым критериям, чтобы оценивать последовательно.**
- [ ] Для каждого трека есть краткое пояснение, что считается хорошим уровнем
- [ ] Подсказки доступны во время выставления оценки
---
**[US-016][Гость] Как гость, я хочу оставить обратную связь о конференции и докладах, чтобы улучшить следующий ивент.**
- [ ] Есть поле «насколько понравилось»
- [ ] Есть поле «самые интересные доклады»
- [ ] Есть поле «что улучшить»
---
**[SS-007][Система] Как система, я хочу собирать обратную связь в день события и после, чтобы повысить отклик.**
- [ ] Доступна форма обратной связи в день события
- [ ] Доступна повторная форма после события

### [EPIC-009] Фоллоу‑ап
---
**[US-017][Все роли] Как пользователь, я хочу отметить “хочу контакт” по проекту, чтобы продолжить общение после события.**
- [ ] Можно отметить “хочу контакт” для выбранного проекта
- [ ] Отметка видна организаторам
---
**[SS-008][Система] Как система, я хочу фиксировать запросы на контакт, чтобы организаторы могли связать стороны.**
- [ ] Запросы на контакт сохраняются
- [ ] Организаторы видят список запросов

## Gap Analysis & Risks

### Пропущенные функции
- [💡 Предложение] Механизм уточнения профиля интересов после первых рекомендаций.
- [💡 Предложение] Сценарий повторного напоминания для тех, кто не подтвердил участие.

### Открытые вопросы
- Нужны ли дополнительные статусы заявок участников помимо «подано» → «подтверждено»?
- Нужна ли детализация полей карточки проекта для разных ролей?

### Риски
- [⚠️ Risk] Низкий отклик на обратную связь снизит ценность для организаторов.
- [⚠️ Risk] Разные роли могут ожидать разные критерии релевантности.
