# Information Architecture: AI-first Unconference Navigator

> Версия: 1.0  
> Дата: 1 февраля 2026  
> Основано на: Brief v1, User Story Map v1, Personas v1

---

## 1. Пользователи и их цели

### Основные роли

| Роль | Главная цель | Частота использования |
|---|---|---|
| Организатор/модератор | Покрыть секции экспертами и участниками, видеть подтверждения | Ежедневно в период подготовки |
| Эксперт/жюри | Быстро найти релевантные проекты и оценить | В дни мероприятия |
| Участник проекта | Подтвердить участие и тему/время | Несколько раз до события |
| Гость/партнёр | Найти 3–5 релевантных проектов и оставить фидбек | В день события |

### Mental Model

- **Организатор:** воспринимает как «панель управления событием» (статусы + покрытие).  
- **Эксперт/участник/гость:** воспринимают как «личный помощник» в Telegram.

## 2. Sitemap/Appmap (Карта страниц)

```
AI-first Unconference Navigator
├── Telegram Bot (пользовательская часть)
│   ├── 🧭 Онбординг и роль
│   ├── 🎯 Профиль интересов
│   ├── ⭐ Рекомендации (3–5 проектов)
│   ├── 📌 Мой список / избранное
│   ├── 🗓 Подтверждение участия (эксперты/участники)
│   ├── 🔔 Напоминания и дедлайны
│   ├── 🧾 Оценки и фидбек
│   ├── 🤝 Хочу контакт (follow‑up)
│   └── ⚙️ Настройки
│
└── Admin Console (организатор)
    ├── 🏠 Дашборд события
    ├── 📊 Покрытие тематик/секций
    ├── ✅ Подтверждения и заявки
    ├── 🧠 Проекты и материалы
    ├── 🗂 Секции и расписание
    ├── ⭐ Оценки экспертов
    ├── 💬 Обратная связь гостей
    ├── 🤝 Запросы контакта
    ├── 🔔 Уведомления
    └── ⚙️ Настройки события
```

### Описание разделов

| Раздел | Назначение | Доступ |
|---|---|---|
| Онбординг и роль | Выбор роли, старт сценария (организатор — по приглашению или назначению) | Все роли |
| Профиль интересов | Теги/ключевые слова для матчинга | Все роли |
| Рекомендации | Список релевантных проектов | Все роли |
| Подтверждение участия | Заявки экспертов/участников | Эксперт/Участник |
| Оценки и фидбек | Оценка проектов/мероприятия | Эксперт/Гость |
| Дашборд события | Общая картина для орг‑команды | Организатор |
| Покрытие тематик | Контроль секций и экспертов | Организатор |
| Подтверждения | Статусы заявок/участия | Организатор |
| Уведомления | Планирование напоминаний | Организатор |

## 3. Глобальная навигация

### Основная навигация (Telegram Bot)

| Позиция | Элемент | Куда ведёт | Приоритет |
|---|---|---|---|
| 1 | Рекомендации | Список релевантных проектов | Высокий |
| 2 | Мой список | Сохранённые проекты | Высокий |
| 3 | Подтверждение | Заявка/подтверждение участия | Высокий |
| 4 | Оценки/Фидбек | Формы оценок | Средний |
| 5 | Настройки | Роль/интересы/уведомления | Средний |

### Основная навигация (Admin Console)

| Позиция | Элемент | Куда ведёт | Приоритет |
|---|---|---|---|
| 1 | Дашборд | Обзор по событию | Высокий |
| 2 | Покрытие | Матрица тематик/секций | Высокий |
| 3 | Подтверждения | Заявки и статусы | Высокий |
| 4 | Проекты | Каталог проектов/материалов | Средний |
| 5 | Оценки/Фидбек | Сводка качества | Средний |
| 6 | Настройки | Параметры события | Низкий |

### Утилитарная навигация

- Поиск: глобальный (проекты, эксперты, секции)
- Уведомления: да (для организатора)
- Помощь: да (FAQ/контакты)

## 4. Ключевые User Flows

### Flow 1: Эксперт подтверждает участие и оценивает проекты

```
Telegram bot → Выбор роли → Подтверждение участия → Напоминание → Оценка проекта → Отправка
```

### Flow 2: Организатор контролирует покрытие

```
Admin Console → Дашборд → Покрытие тематик → Просмотр статусов → Действие (напоминание)
```

### Flow 3: Гость получает рекомендации и оставляет фидбек

```
Telegram bot → Профиль интересов → Рекомендации → Мой список → Фидбек → Follow‑up
```

## 5. Иерархия контента

### Приоритеты отображения

| Приоритет | Контент | Почему важен |
|---|---|---|
| P1 | Статусы подтверждений / покрытие | Ключевая цель организатора |
| P1 | Рекомендации и избранное | Ключевая цель гостей/экспертов |
| P2 | Оценки и фидбек | Важны, но после основного действия |
| P3 | Настройки и справка | Вспомогательное |

### Группировка контента

- **Проекты:** по теме/индустрии/стадии, далее по релевантности  
- **Секции:** по трекам и статусу подтверждения  
- **Оценки:** по проектам и трекам

---

## 6. Поиск и обнаружение

### Поиск

- **Тип:** глобальный (в админке) + локальный в рекомендациях
- **Что ищем:** названия проектов, теги, секции, экспертов
- **Фильтры:** тематика, индустрия, стадия, трек, статус подтверждения
- **Сортировка:** по релевантности (по умолчанию), по времени, по стадии

### Обнаружение контента

- Рекомендации: есть
- Связанные элементы: есть (похожие проекты по тегам)
- Популярное/Недавнее: для организаторов (последние подтверждения)

## 7. Состояния страниц

### Пустые состояния

| Страница | Когда пусто | Что показываем |
|---|---|---|
| Рекомендации | Нет интересов | CTA заполнить профиль |
| Мой список | Нет сохранённых проектов | Подсказка «выбрать из рекомендаций» |
| Покрытие тематик | Нет данных по секциям | Напоминание заполнить секции |

### Состояния загрузки

- Первая загрузка: скелетон (список проектов/матрица)
- Подгрузка данных: спиннер

### Ошибки

- Нет сети: повторить запрос
- Нет доступа: сообщение о роли
- 404: ссылка устарела

### Адаптация навигации

- Telegram: кнопки/команды, короткие ответы
