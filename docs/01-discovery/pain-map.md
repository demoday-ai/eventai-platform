# Pain Map по сегментам

_Структурированная карта болей на основе Customer Discovery интервью с организатором Demo Day._

---

## Сегмент 1: Организатор

### Pain 1: Хаотичное составление расписания

| Параметр | Значение |
|----------|----------|
| **Боль** | Один человек вручную собирает расписание, ChatGPT кластеризует плохо, дубликаты |
| **Severity** | 5/5 |
| **Frequency** | 5/5 (каждый Demo Day) |
| **Reach** | 5/5 (затрагивает всех: 330 студентов + 50 экспертов + оргов) |
| **Confidence** | 5/5 |
| **Priority Score** | 50 |
| **Evidence** | "Она с ChatGPT делала расписание и у неё задублировалось, она это не проверяла"; "В двух комнатах одновременно один проект, а он вообще уже выступил в предыдущий день"; "ChatGPT кластеризует фигово" |
| **Root Cause** | Нет системы: один человек + ChatGPT + Google-таблица. Нет валидации, нет дедупликации |
| **Workaround** | Ручная проверка (которая не делается), ChatGPT для кластеризации (работает плохо) |
| **Desired Outcome** | AI-кластеризация с учётом описаний + GitHub, автоматическая дедупликация, балансировка залов |
| **JTBD** | Когда нужно распределить 330 проектов по 6-10 залам, хочу автоматически кластеризовать и исключить дубликаты, чтобы расписание было готово за дни, а не недели |

### Pain 2: Неподтверждённые студенты

| Параметр | Значение |
|----------|----------|
| **Боль** | 80 студентов не заполнили форму, 15 проектов не пришли, всё выясняется в последний момент |
| **Severity** | 5/5 |
| **Frequency** | 5/5 |
| **Reach** | 4/5 (24% студентов проблемные) |
| **Confidence** | 5/5 |
| **Priority Score** | 48 |
| **Evidence** | "В двенадцать часов ночи восемьдесят человек себя не нашли"; "Пятнадцать проектов — люди просто не пришли, они нам не отвечали" |
| **Root Cause** | Нет системы подтверждения участия. Студенты не заполняют формы, не читают пинги |
| **Workaround** | Массовые пинги в чатах (не работает), экстренное распределение в ночь перед DD |
| **Desired Outcome** | Бот персонально присылает слот, студент подтверждает/отказывается за 2 недели |
| **JTBD** | Когда расписание готово, хочу знать заранее, кто придёт, а кто нет, чтобы не было хаоса в день DD |

### Pain 3: Нехватка и непокрытие экспертами

| Параметр | Значение |
|----------|----------|
| **Боль** | Нужно 60-80 экспертов, есть ~50, активных ~30-40. Часть не приходит, часть молчит |
| **Severity** | 4/5 |
| **Frequency** | 5/5 |
| **Reach** | 3/5 (50 экспертов + те студенты, кто без эксперта) |
| **Confidence** | 5/5 |
| **Priority Score** | 42 |
| **Evidence** | "Эксперты тоже не приходят, они выходят, кто-то просто сидит и молчит, кто-то оценки не выставляет"; "У нас в комнате никто не пришёл" |
| **Root Cause** | Ручное приглашение, нет системы подтверждения, раньше давали выбирать → хаос |
| **Workaround** | Организатор лично пишет каждому, назначает комнату, но не всегда релевантную |
| **Desired Outcome** | Бот предлагает комнату по тегам → эксперт подтверждает → напоминалки |
| **JTBD** | Когда нужно покрыть 6-10 залов экспертами, хочу автоматический матчинг по тегам интересов, чтобы каждый зал был обеспечен |

### Pain 4: Обратная связь не доходит до студентов

| Параметр | Значение |
|----------|----------|
| **Боль** | ОС собирается, но не рассылается. Качество разное: от "Херня!" до конструктива |
| **Severity** | 3/5 |
| **Frequency** | 5/5 |
| **Reach** | 5/5 (330 студентов) |
| **Confidence** | 5/5 |
| **Priority Score** | 40 |
| **Evidence** | "Было бы хорошо, если бы студентам рассылалась обратная связь"; "Кто-то может написать 'Херня!' и всё"; "В саппорт постоянно пишут: можно ОС увидеть?" |
| **Root Cause** | Нет ресурсов на предобработку, неконструктивные комментарии нельзя рассылать как есть |
| **Workaround** | Никак — ОС не рассылается вообще |
| **Desired Outcome** | AI предобрабатывает ОС (фильтрует неконструктив) → орг модерирует → рассылка |
| **JTBD** | Когда Demo Day закончился, хочу автоматически отправить каждому студенту отфильтрованную обратную связь, чтобы они получили пользу от выступления |

### Pain 5: Нет данных в реальном времени

| Параметр | Значение |
|----------|----------|

| **Frequency** | 5/5 |
| **Reach** | 1/5 (5 оргов) |
| **Confidence** | 4/5 |
| **Priority Score** | 30 |
| **Evidence** | "Нету нормальной системы, которая это всё отслеживает" |
| **Root Cause** | Нет инструмента, всё в головах организаторов |
| **Workaround** | Ходят по залам, спрашивают модераторов |
| **Desired Outcome** | Дашборд: кто пришёл, покрытие, пустые слоты, алерты |
| **JTBD** | Во время DD хочу видеть реальную картину по всем залам, чтобы быстро реагировать на проблемы |
