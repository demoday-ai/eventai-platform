# Клиентские пути Demo Day: AS-IS → TO-BE

_На основе аналитики прошлого DD, интервью с организатором и RICE-матрицы._

---

## Роли

| Роль | Кол-во | Приоритет |
|------|--------|-----------|
| Организатор | ~5 | Основной |
| Эксперт / ментор / жюри | ~50 (активных ~30-40) | Основной |
| Студент (автор проекта) | ~330 проектов | Основной |
| Внешний гость | ~50 доходимость (190 регистраций) | Вторичный |
| Бизнес/партнёр | ~15-20 (подмножество гостей) | Вторичный |

## 1. Организатор

### AS-IS (сейчас)

```
За 2-3 недели до DD
│
├─ Собирает формы контрольного рубежа (вручную)
│  └─ Проблема: часть студентов не заполняет
│
├─ Выгружает в ChatGPT для кластеризации по темам
│  └─ Проблема: ChatGPT кластеризует плохо, кто-то написал
│     описание, кто-то — одну строчку, кто-то — ссылку на GitHub
│
├─ Один человек вручную составляет расписание
│  └─ Проблема: дубликаты (проект в двух комнатах),
│     неравномерная нагрузка (зал 2 = 30 проектов, зал 1 = 15)
│
├─ Публикует расписание
│  └─ Проблема: 80 человек себя не нашли (в 12 ночи!)
│     → экстренное распределение
│
├─ Вручную зовёт экспертов (лично пишет каждому)
│  └─ Проблема: нужно 60-80, набирается ~50,
│     активных ~30-40, часть не приходит
│
├─ Назначает экспертов на комнаты
│  └─ Проблема: раньше давали выбирать — эксперт прыгал
│     между комнатами и никого не оценивал
│
День DD
│
├─ Гасит пожары: кто-то не пришёл, слоты пустые,
│  нужно звать замену, эксперты молчат
│  └─ ~15 проектов просто не явились
│
├─ Модерирует зал
│  └─ Проблема: "у нас в комнате никто не пришёл из экспертов,
│     вопросы задавали коллеги"
│
После DD
│
├─ Собирает оценки (вручную из таблиц)
│  └─ Проблема: не все эксперты выставляют, игнорят
│
├─ Хочет разослать обратную связь студентам
│  └─ Проблема: ОС неравномерная ("Херня!" vs конструктив),
│     нужна предобработка, ресурсов нет
│
└─ Собирает ОС от экспертов о DD
   └─ Проблема: ОС касается конкретного DD, не переносится
      в следующий раз, нет системы извлечения инсайтов
```

### TO-BE (dream-сценарий)

```
За 1 месяц до DD
│
├─ Бот собирает данные о проектах
│  ├─ Автоматически тянет из формы контрольного рубежа
│  ├─ Парсит GitHub-репозитории (README, структура, активность)
│  ├─ Напоминает студентам заполнить форму (за месяц, за 2 недели, за неделю)
│  └─ Формирует карточку проекта: название, описание, стек, теги, артефакты
│
├─ AI кластеризует проекты по темам
│  ├─ На основе описаний + GitHub + артефактов (не одной строчки)
│  ├─ Учитывает мультидисциплинарность (22 проекта с 3+ тегами)
│  ├─ Предлагает распределение по залам с балансировкой нагрузки
│  └─ Орг просматривает, корректирует, подтверждает
│
├─ Дедупликация и валидация
│  ├─ Система не даёт поставить проект в два зала
│  ├─ Проверяет: все ли студенты в расписании
│  └─ Отчёт: "80 человек без слота — вот список"
│
За 2 недели до DD
│
├─ Бот рассылает расписание студентам
│  ├─ Персонально: "Ты выступаешь в Зале 3, 14:15-14:30"
│  ├─ Студент подтверждает: "Да, буду" / "Нет, не смогу"
│  └─ Если нет подтверждения — эскалация куратору
│
├─ Бот приглашает экспертов
│  ├─ По табличке тегов: "Сергей, LLM-инженер → Зал 3 (Агенты, NLP)"
│  ├─ Эксперт подтверждает / просит другую комнату
│  ├─ Дашборд: покрытие по комнатам (зелёное/красное)
│  └─ Если комната не покрыта — орг видит и добирает
│
За 1 день до DD
│
├─ Напоминалки всем: студентам, экспертам, модераторам
│  └─ "Завтра Зал 3, 14:00. Вот расписание твоей комнаты"
│
├─ Эксперты получают овервью проектов своей комнаты
│  └─ Краткие карточки: описание, стек, GitHub-статус, ключевые артефакты
│
День DD
│
├─ Напоминалка за час до слота
│
├─ Чекин: студент/эксперт отмечает присутствие
│  └─ Если не отметился за 10 мин — алерт модератору
│
├─ Дашборд реального времени для оргов
│  ├─ Кто пришёл, кто нет
│  ├─ Пустые слоты
│  └─ Покрытие экспертами
│
После DD
│
├─ Автосбор оценок
│  └─ Бот напоминает экспертам выставить оценки (если не выставили)
│
├─ Предобработка обратной связи
│  ├─ AI фильтрует неконструктивное ("Херня!" → не отправляется)
│  ├─ Группирует по темам (техническое, продуктовое, презентация)
│  └─ Орг модерирует → рассылка студентам
│
├─ Аналитика для оргов
│  ├─ Доходимость: % студентов, % экспертов
│  ├─ Покрытие: сколько проектов получили оценку
│  ├─ Качество ОС: средняя длина, конструктивность
│  └─ Инсайты: что улучшить в следующий раз
│
└─ Контакты
   └─ Эксперт просит контакт студента → бот отправляет
      (с согласия студента)
```

## 2. Эксперт / ментор

### AS-IS

```
За 1-2 недели
│
├─ Получает личное сообщение от организатора
│  └─ "Вот ты пойдёшь на эту комнату"
│     (раньше давали выбирать → хаос)
│
├─ Нет информации о проектах заранее
│  └─ Идёт вслепую, не знает, что ожидать
│
День DD
│
├─ Приходит (или нет — ~20% не приходят)
│
├─ Слушает презентации
│  └─ Качество разное: кто-то 2 слайда, кто-то полноценный research
│     Эксперт не знает заранее, что уровень будет "никакой"
│
├─ Задаёт вопросы (или молчит)
│  └─ "У нас эксперты сидели и ничего не говорили"
│
После DD
│
├─ Выставляет оценки (или нет)
│  └─ Проблема: игнорят, забывают, некоторые вообще не оценивают
│
├─ Хочет контакт студента → пишет организатору
│  └─ Организатор вручную скидывает
│
└─ Даёт ОС о DD (если попросят)
   └─ ОС не переносится в следующий раз
```

### TO-BE

```
За 2 недели
│
├─ Бот: "Рустем, приглашаем на DD! По твоим интересам
│  (Agents, LLM, NLP) подходит Зал 3. Вот 12 проектов.
│  Какие слоты тебе удобны?"
│  ├─ Выбор временных слотов (совмещение с работой)
│  ├─ [Да, иду] → фиксируется
│  ├─ [Хочу другую комнату] → варианты
│  └─ [Не смогу] → орг ищет замену
│
За 1 день
│
├─ Бот присылает овервью комнаты:
│  ├─ 12 проектов, по каждому карточка:
│  │  ├─ Название, автор, описание (2-3 предложения)
│  │  ├─ Стек и теги
│  │  ├─ GitHub: есть/нет, активность, README
│  │  └─ Артефакты: презентация, демо
│  └─ Расписание: кто во сколько выступает
│
├─ Напоминалка: "Завтра, Зал 3, 14:00-19:30"
│
День DD
│
├─ За час: "Через час — Зал 3. Первый проект: X"
│
├─ Чекин в боте
│
├─ Слушает, задаёт вопросы
│  └─ Контекст из овервью помогает задавать релевантные вопросы
│     (эксперт уже использует Claude для погружения в контекст —
│      овервью от бота заменяет этот ручной шаг)
│
После DD
│
├─ Бот после каждого проекта: "Оцени проект X"
│  ├─ Оценка по критериям (не просто 1-5)
│  ├─ Автотрекинг пропущенных студентов (не сбивает нумерацию)
│  ├─ Напоминание через день, если не оценил
│  └─ "Если упаковать в пару сообщений, то удобно"
│
├─ Бот: "Хочешь связаться с автором проекта X?"
│  └─ [Да] → контакт (с согласия студента)
│
└─ Бот: "Как прошёл DD? Что улучшить?"
   └─ Короткий опрос (3 вопроса), AI извлекает инсайты
```

## Как должен измениться сам процесс Demo Day

### Структурные изменения

| Сейчас | Должно быть |
|--------|-------------|
| Расписание составляет 1 человек вручную + ChatGPT | AI-кластеризация с ручной модерацией орга |
| Экспертов зовут лично, в лички | Бот приглашает по тегам, эксперт подтверждает |
| Нет подтверждения участия | Бот собирает подтверждения за 2 недели |
| Нет напоминалок | За день, за час — всем ролям |
| Нет овервью проектов для экспертов | Карточки проектов за день до DD |
| Обратная связь не доходит до студентов | Предобработанная ОС рассылается в тот же день |
| Нет данных о доходимости | Дашборд: кто пришёл, кто нет, покрытие |
| Контакты — через организатора вручную | Бот организует с согласия обеих сторон |
| Demo-Hero для гостей (не работает) | Единый бот для всех ролей |
| Формы заполняют "когда-нибудь" | Бот напоминает за месяц, за 2 нед, за неделю |

### Таймлайн нового процесса

```
-30 дней  Бот начинает собирать карточки проектов (из формы + GitHub)
          Напоминалки студентам

-14 дней  AI кластеризует проекты → орг проверяет → расписание
          Бот приглашает экспертов по тегам
          Бот рассылает слоты студентам → подтверждение

 -7 дней  Внешним гостям: "Что ищете?" → персональная программа
          Эскалация неподтверждённых (студенты, эксперты)

 -1 день  Овервью проектов экспертам
          Напоминалки всем

  День DD  Чекин, напоминалки за час
           Дашборд реального времени для оргов
           Алерты: пустые слоты, неявки

 +0 дней  Бот собирает оценки (напоминает если не выставили)

 +1 день  Предобработка ОС → модерация оргом → рассылка студентам
          Связывание гость↔студент по запросу

 +7 дней  Аналитика для оргов: доходимость, покрытие, NPS
          Follow-up гостям
```

### Что НЕ менять

- Формат залов и параллельных сессий (это образовательная структура)
- Роль модератора в зале (человек, не бот)
- Процесс оценивания (руководитель + эксперт)
- Прожарка vs бизнес-презентации (разные форматы для разных задач)

### Что убрать

- Ручную кластеризацию через ChatGPT → AI в системе
- Ручное приглашение экспертов в лички → бот
- Demo-Hero → заменяется единым ботом
- Ручную рассылку ОС → автоматизация с предобработкой

│
├─ Партнёр не может сам ориентироваться
│  ├─ "Функциональный руководитель не может найти, нащупать
│  │   свою проблематику"
│  └─ "Начинается технический доклад — ничего не понимаю, отключаюсь"
│
├─ Отправляет делегатов (2-3 чел) в конкретные комнаты
│  └─ Нужны чёткие инструкции: куда, когда, какие проекты
│
День DD
│
