# RICE-матрица: приоритизация гипотез и фич

_По итогам Customer Discovery интервью #1 (организатор) и аналитики прошлого Demo Day._

## Методология

- **Reach** — сколько пользователей затронет за один Demo Day
- **Impact** — влияние на ключевую метрику (0.25 = минимум, 0.5 = низкий, 1 = средний, 2 = высокий, 3 = массивный)
- **Confidence** — уверенность в гипотезе (100% = подтверждено данными/цитатой, 80% = сильные сигналы, 50% = предположение)
- **Effort** — трудозатраты в person-неделях
- **RICE Score** = (Reach × Impact × Confidence) / Effort

## MVP-скоуп (по RICE)

Исходя из приоритизации, MVP должен включать:

**Must have (RICE > 200):**
1. Напоминалки студентам и экспертам
2. Подтверждение участия студентов
3. Автокластеризация проектов
4. Предобработка и рассылка обратной связи

**Should have (RICE 50-200):**
5. Единая точка входа с ролями (архитектурная основа)
6. Распределение экспертов по комнатам

**Could have (RICE < 50):**
7. Овервью проектов с GitHub
8. Персональная программа для гостей

**Won't have (отклонено/отложено):**
- Q&A-помощник (эксперты не хотят)
- Follow-up пакет с LOI (нет запроса)

## Риски и допущения

- RICE-оценки основаны на **одном интервью** с организатором. Нужна валидация с Лерой (расписание) и Олегом (бизнес-гости)
- Reach для внешних гостей может вырасти, если формат DD изменится
- Effort — грубые оценки, уточнить после проектирования архитектуры
- Confidence по H8/H11 может вырасти после интервью с Олегом (бизнес-сегмент)
