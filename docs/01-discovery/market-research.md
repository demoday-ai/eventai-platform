## Executive Summary

Рынок event-технологий ($12.5–15.3B глобально, CAGR 15–18%) активно растёт, но **ниша управления образовательными Demo Day с сотнями проектов остаётся незакрытой**. Хакатон-платформы (Devpost, Devfolio) решают задачи подачи и судейства, конференц-платформы (Whova, Brella, Swapcard) — нетворкинг для корпоративных мероприятий ($5K–50K/событие). Ни один существующий продукт не объединяет: (a) AI-кластеризацию проектов по залам, (b) матчинг экспертов по тегам, (c) подтверждение участия студентов, (d) персональные программы для гостей, (e) предобработку обратной связи — в едином Telegram-боте. На российском рынке вакуум: западные SaaS недоступны, Timepad/Leader-ID — только регистрация, AI-инструментов для мероприятий нет. Импортозамещение создаёт окно возможностей.

### Детальный анализ топ-3 конкурентов

#### Devpost
- **Описание:** Доминирующая глобальная платформа для хакатонов. Участники подают проекты (описание, код, демо), судьи оценивают. Портфолио проектов — резюме разработчика.
- **Позиционирование:** "Where the world's best developers build tomorrow" — фокус на разработчиков и портфолио
- **Что делают хорошо:** Крупнейшее сообщество, удобный workflow судейства, GitHub-интеграция
- **Что делают плохо:** Нет управления расписанием/залами для многоформатных мероприятий, нет AI-матчинга, нет Telegram
- **Уроки для нас:** Devpost решает *подачу и судейство*, но не *операционную логистику* распределения 330 проектов по залам и матчинга экспертов. Наш бот закрывает именно этот слой.

#### Whova
- **Описание:** All-in-one платформа управления конференциями. Мультитрековая повестка, персональное расписание, AI-матчинг участников, Q&A, аналитика.
- **Позиционирование:** Полный жизненный цикл конференции (регистрация → повестка → нетворкинг → аналитика)
- **Что делают хорошо:** Ближайший аналог по функциям *персонализации для участников*, AI-матчинг, персональное расписание
- **Что делают плохо:** Нет автоматизации *со стороны организатора* (расписание спикеров/проектов по залам — вручную), нет Telegram, дорого ($5K–50K), overkill для образовательных DD
- **Уроки для нас:** Whova подтверждает спрос на персонализацию повестки, но мы решаем проблему шире — включая организаторскую логистику (кластеризация, матчинг экспертов). И делаем это в Telegram, а не в проприетарном приложении.

#### Demo-Hero (@jugru_conf_bot)
- **Описание:** Telegram-бот для навигации гостей на Demo Day AI Talent Hub. Tinder-механика: показывает пары докладов, выбираешь. Формирует маршрут.
- **Позиционирование:** Геймифицированная навигация по мероприятию
- **Что делают хорошо:** Правильный канал (Telegram), попытка решить проблему навигации
- **Что делают плохо:** Tinder-механика не масштабируется (53+ мин на 212 проектов попарно), поверхностное профилирование (пол + "чем занимаешься"), маршрут с багами (пустой), нет организаторских функций, нет follow-up, нет подтверждения участия
- **Уроки для нас:** Bot-first — правильный подход. Но AI должен заменить ручную механику (tinder-пары), а не геймифицировать её. Организатор — главный заказчик, не гость.

### 1.2 Российские event-платформы

#### Timepad
- **URL:** https://timepad.ru
- **Описание:** Ведущая российская платформа для регистрации и продажи билетов на мероприятия. Аналог Eventbrite для российского рынка.
- **Возможности:** Создание мероприятий, лендинги, регистрация, билеты (свободные и платные), CRM участников, email-рассылки, аналитика, интеграция с российскими платёжными системами (Тинькофф, Сбербанк)
- **Цена:** Бесплатно для бесплатных мероприятий; платные: 5% комиссия (или фиксированные тарифы от ~5000 руб/мес)
- **Сильные стороны:** Доминирует в России для регистрации, поддержка российских платежей, известный бренд
- **Слабые стороны:** Только регистрация/билеты — нет управления контентом, расписанием, нет AI, нет Telegram-бота
- **Релевантность:** Timepad — вероятный инструмент AI Talent Hub для регистрации на DD. Решает вопрос "кто зарегистрировался", но ничего об операционной сложности мероприятия.

#### Leader-ID
- **URL:** https://leader-id.ru
- **Описание:** Национальная платформа для мероприятий в российской инновационной/образовательной экосистеме (НТИ, Кружковое движение, Сириус).
- **Возможности:** Профили пользователей с тегами компетенций, календарь мероприятий по России, регистрация и учёт посещений, "цифровой след" активностей, интеграция с Точками Кипения
- **Сильные стороны:** Большая база в российском edu-секторе, профили компетенций, государственная поддержка
- **Слабые стороны:** Бюрократический UX, медленная разработка, нет AI-функций для матчинга/расписания, нет Telegram-интеграции, жёсткость государственной платформы
- **Релевантность:** Leader-ID показывает, что матчинг по компетенциям существует в российской экосистеме, но он на уровне профилей, не на уровне мероприятий. Наш матчинг экспертов по тегам — более гранулярная версия.

### 1.4 AI-ассистенты и чат-боты для мероприятий

- **Zenus (intui.io):** AI-аналитика лиц для мероприятий — измерение вовлечённости, подсчёт посещаемости. Касательный продукт.

**Ни один существующий AI-ассистент для мероприятий не решает задачу автоматизации операционной логистики Demo Day.**

## 2. Косвенные конкуренты и текущие решения

### Как решают проблему сейчас:

#### Google Sheets + ручная работа (Status Quo)
- Организатор создаёт таблицы со списками проектов, вручную назначает залы, лично пишет экспертам в Telegram, ведёт подтверждения в отдельном листе
- **Используется:** AI Talent Hub (подтверждено в CustDev)
- Плюсы: бесплатно, гибко, привычно
- Минусы: не масштабируется (330 проектов = хаос), ошибки (дубликаты, 80 студентов не в расписании), single point of failure, нет автоматизации, нет персонализации
- **Затраты времени:** организатор тратит *дни* на ручное составление расписания

#### ChatGPT для кластеризации
- Организатор вставляет описания проектов в ChatGPT, просит кластеризовать по тематикам, вручную корректирует
- **Используется:** AI Talent Hub (подтверждено: "ChatGPT кластеризует фигово")
- Плюсы: быстрый первый проход по тематической группировке
- Минусы: нестабильное качество, нет работы с ограничениями (ёмкость залов, слоты, доступность экспертов), нет интеграции с данными мероприятия
- **Валидация:** организаторы *уже пробуют AI для расписания*, но нужно целевое решение

#### Notion / Airtable
- Детали мероприятия в базах Notion/Airtable; карточки проектов с тегами; шаринг с командой
- Плюсы: гибкость, совместная работа, реляционные данные
- Минусы: нет автоматизации, нет участник-ориентированного интерфейса, нет уведомлений, всё вручную

#### Telegram-боты для мероприятий (generic)
- Простые Telegram-боты для RSVP (@EventBot, @RSVPbot) — регистрация и базовые напоминания
- Плюсы: Telegram-нативные, простые
- Минусы: нет AI, нет матчинга, нет расписания, не для мультитрековых мероприятий с сотнями сессий

#### Платформы автоматизации (n8n, Zapier, Make)
- **n8n** (https://n8n.io) — open-source автоматизация workflow. Может связать Telegram, Google Sheets, AI API
- **Zapier** (https://zapier.com) — облачная автоматизация. Триггеры и действия между приложениями
- Плюсы: гибкие building blocks, можно собрать часть решения
- Минусы: строительные блоки, не готовый продукт. Команде нужно проектировать и строить всю логику управления мероприятием

### Adjacent markets:
- **Академический рецензионный матчинг** — Toronto Paper Matching System (TPMS), используется OpenReview для NeurIPS, AAAI. Автоматический матчинг рецензентов к статьям по тематическому сходству. Концептуально близко к нашему матчингу экспертов по тегам, но не доступно как продукт для мероприятий.
- **Системы управления обучением (LMS)** — Moodle, Canvas. Управляют учебным процессом, но не мероприятиями.
- **Системы управления клинтами (CRM)** — AmoCRM, Bitrix24. Управляют контактами, но не логистикой мероприятий.

| **SOM** (Year 1–2) | ~$100K–450K | Российские AI-программы + акселераторы + хакатоны | 🔴 Требует проверки |
| **SOM** (Year 3–5, с PMF) | ~$1–3M | Расширение на EdTech-программы | 🔴 Требует проверки |

### Комментарий к оценкам:
> TAM включает все инструменты для мультитрековых мероприятий с проектными презентациями. SAM ограничен AI-инструментами для образовательных DD/хакатонов на рынках с высоким проникновением Telegram (Россия, СНГ, ЦА). SOM Year 1–2 — реалистичная оценка: AI Talent Hub ($5–10K) + 10–20 аналогичных программ (ВШЭ, ИТМО, Иннополис, Сколтех, Сириус, НТИ) × $5–10K + 20–50 российских хакатонов × $3–5K.

## 5. Best Practices и тренды

### Индустриальные стандарты для крупных Demo Day (100+ проектов)

**Расписание:**
- Кластеризация проектов по тематике, а не по алфавиту или случайно
- Минимизация конфликтов между треками по популярным темам (наши данные: NLP во всех 6 залах — катастрофа)
- Оптимизация с ограничениями: доступность экспертов, предпочтения команд, ёмкость залов
- Публикация расписания за 48+ часов (а не в ночь перед DD)

**Управление экспертами:**
- Подтверждение за неделю, повторное подтверждение за день
- Матчинг по тегам + личным предпочтениям
- Минимум 2 эксперта в зале (резерв для no-show)
- Рассылка подготовительных пакетов: резюме проектов, критерии оценки

**Навигация участников:**
- Персональные расписания (не только общая повестка)
- Push-уведомления для предстоящих сессий, изменений залов
- Поиск/фильтр по тематике, технологии, отрасли

**Обратная связь:**
- Сбор ОС во время мероприятия (а не только после)
- Предобработка и фильтрация перед рассылкой студентам
- Отправка ОС в течение 48 часов (а не через недели)

**Follow-up:**
- Механизм обмена контактами
- Саммари-пакеты (что видел, кто выступал, контакты)
- Шаблоны follow-up писем
- Метрика пост-event engagement

### Формирующиеся тренды

#### AI в управлении мероприятиями (2024–2026)
- **AI-расписание и оптимизация:** Конференции всё чаще используют AI для составления расписания сессий. Методы из operations research (constraint satisfaction, ILP) применяются к планированию конференций. NeurIPS и AAAI используют TPMS для автоматического матчинга рецензентов.
- **AI-матчинг:** Brella, Grip, Swapcard вложились в AI-матчинг (2023–2025). Тренд: от keyword-матчинга к семантическому (embeddings). Наше использование LLM для поиска проектов — state-of-the-art для этого пространства.
- **AI-генерация контента:** Платформы автогенерируют саммари сессий, биографии спикеров, гайды. Наши "брифы проектов" и "Q&A-подсказки" вписываются в этот тренд.
- **AI-ассистенты мероприятий:** Чат-интерфейсы для мероприятий появляются (2024–2025). Большинство — FAQ-боты, но тренд к проактивным персонализированным ассистентам.

#### Chatbot-first подход
- WhatsApp Business API и Telegram Bot API позволяют управлять мероприятиями без отдельных приложений
- На рынках с высоким проникновением Telegram (Россия, СНГ, Иран, часть ЮВА) Telegram-нативные решения имеют преимущество
- Ключевой инсайт CustDev: "Я бы не пользовалась никаким приложением" — подтверждение, что Telegram — единственный канал
- Тренд ускорился с ChatGPT/LLM adoption (пользователи привыкли к conversational AI)

#### Гиперперсонализация
- Участники ожидают Netflix-like рекомендации ("мероприятия для вас")
- Сдвиг от "вот полная повестка" к "вот ВАША повестка"
- Наша "персональная программа" — в русле этого тренда

#### Post-event engagement
- Растущий акцент на follow-up (ранее игнорировался). Мероприятия с follow-up пакетами (контакты, саммари, next steps) показывают 2–3× удовлетворённость
- Наш "follow-up пакет" с контактами, шаблонами LOI и next steps — дифференциатор

#### Тренды российского рынка
- **Telegram как инфраструктура:** 100M+ MAU в России (2025). Дефолтный канал для tech, образования, бизнеса. Компании строят бизнес-инструменты как Telegram-ботов
- **AI-event бум:** AI-экосистема России быстро растёт. Новые конференции, хакатоны и DD появляются ежемесячно
- **Импортозамещение:** Организации ищут альтернативы западным SaaS (нет Whova, нет Swapcard, нет Cvent). Чёткая рыночная возможность для российских решений

### Технологии, меняющие рынок:
- **LLM с function calling и tool use** — позволяют строить сложных AI-агентов для мероприятий
- **Semantic search (embeddings)** — замена keyword-матчинга на семантический для рекомендаций проектов
- **Telegram Bot API расширения** — Web Apps, inline mode, payment API
- **Constraint optimization solvers** — для автоматического расписания с ограничениями

### UX best practices для Telegram-ботов мероприятий
- Онбординг < 2 минут. 3–5 целевых вопросов, не 20
- Никаких tinder-механик для 100+ элементов. Поиск/фильтр + AI-рекомендация
- Проактивные уведомления, не только реактивные. "Через 15 мин — твоя сессия, Зал 3"
- Inline-кнопки для структурированных выборов; свободный текст для открытых запросов
- Персистентный контекст: бот помнит предыдущие взаимодействия
- Graceful degradation: если AI-рекомендация провалилась — показать фильтрованный список

## 7. Gaps & Opportunities

### Незакрытые ниши:
1. **Автоматическое расписание проектов по залам:** Ни один конкурент не предлагает AI-кластеризацию проектов по комнатам с учётом ограничений (balance, дедупликация, тематическая когерентность)
2. **Матчинг экспертов по тегам для мероприятий:** Ближайший аналог — академический paper-reviewer matching (TPMS), но он не доступен как продукт для мероприятий
3. **Telegram-нативная event-платформа:** На рынках с высоким проникновением Telegram — вакуум
4. **Образовательные Demo Day как сегмент:** Конференц-инструменты рассчитаны на 1,000+ участников, хакатон-платформы — на соревнования. Образовательные витрины с 100–330 проектами — между стульями

### Возможности для дифференциации:
- **Организатор-first подход:** Конкуренты фокусируются на участниках; наш CustDev подтвердил, что организатор — главный заказчик
- **Полный workflow в одном боте:** End-to-end: расписание → подтверждение → персонализация → ОС → follow-up. Нигде не собрано вместе
- **Правильная цена:** $0–10K vs $5K–50K у enterprise-инструментов
- **Доменная экспертиза:** Команда встроена в экосистему AI Talent Hub

### Потенциальные Blue Ocean:
- **Российский образовательный event-tech:** Нет ни одного специализированного российского продукта для управления Demo Day
- **SaaS для акселераторов:** 30–50 Demo Day/год в России × $3K–10K = рынок $90K–500K, растущий
- **Расширение на постерные сессии:** Академические конференции (NeurIPS-формат) с сотнями постеров — та же проблема навигации
