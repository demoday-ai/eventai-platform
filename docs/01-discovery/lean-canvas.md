_Обновлён по итогам Customer Discovery: интервью #1 (организатор Инна), #2 (эксперт Рустем), #3 (Олег Шатов, руководитель Хаба / бизнес-партнёры), #4 (Анастасия Гапеева, гость-абитуриент), #5 (Дмитрий Ботов, автор идеи / организатор), ТЗ автора идеи (Д. > **Версия:** 2.0
> **Дата:** 2026-02-02

## 1. Проблема

**Топ-3 проблемы (по CustDev):**
1. **Хаос расписания:** составляется вручную одним человеком, ChatGPT кластеризует проекты плохо, дубликаты в расписании (один проект в двух комнатах), 80 студентов не попали в расписание за ночь до Demo Day
2. **Непокрытие экспертами:** нужно 60-80 экспертов, активных ~30-40, часть не приходит, не ставит оценки, молчит. Распределение по комнатам вручную, раньше давали выбирать — эксперт выбирал из разных комнат и в итоге не оценивал никого
3. **Нет системы подтверждения и напоминаний:** ~15 проектов не пришли, эксперты тоже не приходят, студенты не заполняют формы — всё выясняется в последний момент

**Проблемы внешних гостей (подтверждено интервью #4):**
- Гость видит <20% проектов (34 из 212 при 6 параллельных залах)
- "Гигантская гугл-таблица, в которой чёрт ногу сломит" (Настя) — расписание непонятно
- Пропускает самое интересное из-за сдвигов тайминга
- Demo-Hero — "фигня"

**Проблемы бизнес/партнёров (подтверждено интервью #3):**
- Олег вручную подбирает проекты для каждого партнёра — бутылочное горлышко
- Бизнес "не находит свою проблематику" — "технический доклад, я ничего не понимаю, отключаюсь"
- "Каннибализируем свой собственный трафик" — крутые проекты остаются незамеченными
- Post-DD: партнёры запрашивают нарезки из Zoom, презентации — делается вручную

**Существующие альтернативы:**
- Ручное составление расписания (один человек + ChatGPT для кластеризации)
- Ручное распределение экспертов по комнатам (организатор знает экспертов лично)
- Demo-Hero (@jugru_conf_bot) — "фигня", "максимально фиговый", студенты делали
- Таблица с тегами экспертов (ручной матчинг)
- Олег лично подбирает проекты для бизнес-партнёров

## 2. Сегменты пользователей

**Основные (по CustDev — 99% аудитории Demo Day):**
- **Организаторы** (core-команда Хаба, ~5 чел) — составляют расписание, распределяют экспертов, гасят пожары
- **Эксперты / менторы / жюри** (~50 чел, активных ~30-40) — оценивают проекты, задают вопросы
- **Студенты** (~330 проектов) — выступают, получают обратную связь

**Вторичные (но обязательные для продукта):**
- **Бизнес/партнёры** (~15-20 чел) — приходят через нетворк Олега, ищут проекты/технологии/кадры
- **Внешние гости** (~50 реальных из 190 регистраций):
  - 86 работают с ИИ
  - 56 рассматривают обучение (абитуриенты)
  - 31 без конкретной роли
  - 7 представляют бизнес
  - 3 инвестируют

## 3. Уникальное ценностное предложение

AI-платформа (Telegram-бот) — единая точка входа для всех 5 ролей Demo Day. Автоматизирует полный жизненный цикл мероприятия: от кластеризации проектов и распределения экспертов до персональной программы для гостей и follow-up пакетов.

**High-level concept:** "Единая точка входа для всех ролей Demo Day в Telegram"

## 4. Решение

**Для организаторов:**
1. **Автокластеризация проектов** — по данным контрольного рубежа + GitHub (вместо ChatGPT), дедупликация, проверка полноты
2. **Распределение экспертов** — матчинг по тегам интересов + выбор удобных слотов, подтверждение через бота
3. **Дашборд покрытия** — кто пришёл, кто нет, где нет эксперта, какие слоты пустые

**Для экспертов:**
4. **Персональное приглашение** — бот предлагает комнату по интересам + выбор слотов, эксперт подтверждает
5. **Овервью проектов** — краткий бриф по каждому проекту в комнате (с учётом GitHub)
6. **Напоминалки** — за день, за час

**Для студентов:**
7. **Подтверждение участия** — бот присылает слот, студент подтверждает или отказывается
8. **Рассылка обратной связи** — предобработанная (фильтрация неконструктивного) обратная связь от экспертов

**Для внешних гостей (подтверждено интервью #4):**
9. **Персональная программа** — профилирование (свободный текст + варианты) → подборка проектов. Настя: "введя свои интересы, мне предложил бот доклады — было бы очень удобно" (5/5)
10. **Follow-up пакет** — конспект просмотренных проектов, рекомендации (Настя: 5/5)
11. **Подсказки вопросов** — "с поддержкой побольше задавала" (3/5)
12. **Уведомления о сдвигах тайминга** — "если бы я знала, что тайминг съехал, я бы ушла"

**Для бизнес/партнёров (подтверждено интервью #3):**
13. **Профилирование** — задача (инвестиции / найм / технология / партнёрство) + отрасль/стек → бизнес-подборка проектов
14. **Q&A-помощник** — AI с контекстом проектов. Олег: "сто процентов не в контексте, не могу ответить"
15. **Follow-up пакет** — контакты авторов (с согласия), нарезки презентаций, next steps, шаблоны LOI

## 5. Каналы

- **Telegram-бот** (единственный приемлемый канал — Инна: "Я бы не пользовалась никаким приложением"; Настя: "Телеграмм")
- Рассылка от организаторов
- Личные приглашения через нетворк Олега (для бизнес-партнёров)
- Интеграция с формой контрольного рубежа (данные о проектах)

- Разработка и поддержка
- Интеграция с источниками данных (форма контрольного рубежа, GitHub)

## 8. Ключевые метрики

**Организаторы:**
- Время составления расписания (сейчас: дни вручную → цель: часы)
- % студентов подтвердивших участие до Demo Day
- % экспертов покрывших все слоты в своей комнате
- Количество дубликатов/ошибок в расписании

**Эксперты:**
- Конверсия: приглашение → подтверждение → присутствие
- % оценок выставленных по итогам

**Студенты:**
- % студентов получивших обратную связь
- Удовлетворённость обратной связью

**Внешние гости:**
- Конверсия: зашёл в бота → получил программу → пришёл на рекомендованные проекты
- NPS, % follow-up пакетов открытых

**Бизнес/партнёры:**
- Конверсия: профилирование → подборка → контакт с автором
- % партнёров получивших follow-up пакет
- Количество инициированных контактов (бизнес → студент)

## 9. Нечестное преимущество

- Прямой доступ к данным проектов и процессам AI Talent Camp (мы команда кэмпа)
- AI-first подход vs. ручной процесс + ChatGPT
- Знание реальных болей из первых рук (4 CustDev-интервью с разными сегментами)
- Единая система вместо разрозненных Google-форм и таблиц
- Нет аналогов на российском рынке (Telegram-нативная event-платформа для DD)
