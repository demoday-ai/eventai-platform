# Ideal Customer Profile (ICP) по сегментам

_На основе Customer Discovery: интервью #1 (организатор Инна), #2 (эксперт Рустем), #3 (Олег Шатов, бизнес-партнёры), #4 (Анастасия Гапеева, гость-абитуриент), аналитики прошлого Demo Day._

---

## Сегмент 1: Организатор Demo Day

| Поле | Описание |
|------|----------|
| **Customer Type** | B2B (внутренний заказчик — AI Talent Hub) |
| **Должность** | Сотрудник AI Talent Hub: координатор проектного трека, модератор, продюсер мероприятия |
| **Демография** | 22-35 лет, высшее образование (IT/менеджмент), опыт организации мероприятий |
| **Размер команды** | Core-команда ~5 человек, из них 1-2 напрямую занимаются расписанием |
| **География** | Москва / Санкт-Петербург, офлайн + удалённо |
| **Психография** | Перегружены рутиной, ценят автоматизацию, устали от хаоса. "Формат устаревший", "с каждым разом как будто хуже". Прагматики — хотят, чтобы "просто работало" |
| **Поведение** | Работают в Telegram (единственный канал), Google-формы, Google-таблицы, ChatGPT для кластеризации. Не хотят новые приложения: "Я бы не пользовалась никаким приложением" |
| **Потребности** | Автоматизация расписания, система подтверждений, покрытие экспертами, предобработка обратной связи |
| **Триггеры покупки** | Приближение следующего Demo Day (за 1-2 месяца), рост числа проектов, уход ключевых сотрудников |
| **Процесс принятия решений** | Решение принимает руководство Хаба (Дмитрий Ботов). Организатор — конечный пользователь и внутренний чемпион |
| **Бюджет** | Внутренний проект, бюджет = ресурсы команды кэмпа. API-расходы минимальны |
| **Каналы** | Telegram (личные сообщения, групповые чаты). Не email, не приложения |
| **Возражения** | "Студенты всё равно не будут заполнять", "Насильно не заставишь", скепсис к новым инструментам после Demo-Hero |
| **Как преодолеть** | Показать работающий прототип на реальных данных, интеграция с существующей формой контрольного рубежа, минимум лишних действий |

**Ключевая цитата:** "Ребята, это пипец! Если вы как-то упростите этот ужас, я буду вам благодарна."

## Сегмент 2: Эксперт / ментор / жюри

| Поле | Описание |
|------|----------|
| **Customer Type** | B2C (индивидуальный эксперт, приглашается персонально) |
| **Должность** | ML/NLP/CV инженер, исследователь, продакт-менеджер, предприниматель в AI |
| **Демография** | 25-45 лет, профильное техническое образование, 3+ лет опыта |
| **Размер пула** | ~50 приглашённых, активных ~30-40. Нужно 60-80 для полного покрытия |
| **География** | Москва / Санкт-Петербург / удалённо |
| **Психография** | Ценят своё время. Мотивация: менторство, нетворкинг, поиск талантов. Раздражает некачественная подготовка студентов. Не хотят "прыгать между комнатами" |
| **Поведение** | Подтверждают участие лично организатору в Telegram. Часть не приходит (~20%), часть не оценивает, часть молчит на защитах |
| **Потребности** | Знать заранее, что ожидать (овервью проектов). Быть назначенным на релевантную комнату. Простой процесс оценки |
| **Триггеры** | Личное приглашение от знакомого организатора, интересная тематика комнаты |
| **Процесс принятия решений** | Индивидуальный. Решает сам, основываясь на интересе и загрузке |
| **Бюджет** | Волонтёрская роль (бесплатно). Цена = потраченное время |
| **Каналы** | Telegram (личные сообщения от организатора) |
| **Возражения** | "Зачем я пришёл, если тут только продуктовое / только техническое?", "Предложения по вопросам — нет, это от человека должно исходить" |
| **Как преодолеть** | Матчинг по тегам интересов, овервью проектов заранее, простая оценка в боте |

**Ключевая цитата:** "Зачем я вообще пришел, потратил свое время?" (эксперт после нерелевантной комнаты)

## Сегмент 3: Студент (автор проекта)

| Поле | Описание |
|------|----------|
| **Customer Type** | B2C (студент AI Talent Hub) |
| **Должность** | Студент 1-2 курса магистратуры, автор/участник проекта |
| **Демография** | 20-28 лет, техническое образование (CS/DS/ML), работают индивидуально или в команде (2-4 чел) |
| **Размер аудитории** | ~330 проектов на Demo Day |
| **География** | Москва / Санкт-Петербург / регионы (онлайн-участие) |
| **Психография** | Загружены учёбой, проект — не всегда приоритет. "Студентам пофиг" (цитата организатора). Ценят обратную связь, но не получают её. Хотят быть замеченными |
| **Поведение** | Заполняют формы в последний момент (80 не заполнили вовремя), не проверяют расписание, ~15 проектов не пришли. Качество подготовки разное: "кто-то 2 слайда, кто-то реально исследование" |
| **Потребности** | Знать свой слот заранее, подтвердить участие, получить обратную связь после выступления |
| **Триггеры** | Дедлайн контрольного рубежа, приближение Demo Day, напоминалки |
| **Процесс принятия решений** | Вынужденное участие (часть учебного процесса). Мотивация: оценка + ОС от экспертов |
| **Бюджет** | Нет (студенты). Они — пользователи, не покупатели |
| **Каналы** | Telegram (групповые чаты курса, боты) |
| **Возражения** | Не заполняют формы, не реагируют на пинги, не готовят презентации. "Сколько бы не выкладывали критерии, все делают на пофиг" |
| **Как преодолеть** | Напоминалки через бота (персональные, не общие пинги), простые формы, автоматический парсинг GitHub вместо ручного заполнения |

**Ключевая цитата:** "В двенадцать часов ночи восемьдесят человек себя не нашли, потому что они не заполнили форму"

## Сегмент 4: Внешний гость (подтверждено интервью #4)

| Поле | Описание |
|------|----------|
| **Customer Type** | B2C (индивидуальные участники: абитуриенты, AI-практики, другое) |
| **Роли** | Работающие с AI (86), рассматривающие обучение (56), без роли (31). HR — практически нет |
| **Демография** | 18-35 лет, разнородная аудитория. Типичный гость-абитуриент: студент 1 курса, рассматривает поступление |
| **Размер аудитории** | 190 регистраций, реальная доходимость — ~50 (оценка) |
| **География** | Москва / онлайн |
| **Психография** | Любопытство, поиск знаний/образования. Готовы потратить 5-10 мин на профилирование. Ценят персональные рекомендации |
| **Поведение** | Узнают через образовательные каналы ("на курсе сказали"). Теряются в расписании — "гигантская гугл-таблица, в которой чёрт ногу сломит" (Настя). Ходят между залами, пропускают интересное из-за сдвигов тайминга |
| **Потребности** | Персональная программа (5/5), follow-up конспект (5/5), подсказки вопросов (3/5), уведомления о сдвигах. Не нужны формальные встречи 1:1 (1/5) |
| **Триггеры** | Рассылка от Хаба, пост в образовательном канале, рекомендация |
| **Процесс принятия решений** | Спонтанный. Решение прийти — за 1-2 дня |
| **Бюджет** | Бесплатное мероприятие. Ценность = потраченное время |
| **Каналы** | Telegram (бот, предпочтителен). Взаимодействие: за неделю до DD + во время DD |
| **Возражения** | "Расписание непонятное", "не успела посмотреть самый интересный доклад" |
| **Как преодолеть** | Персональная программа в боте (варианты + свободный текст → подборка проектов), уведомления о сдвигах тайминга |

**Ключевая цитата:** "Введя свои интересы, пожелания, мне предложил бот доклады — было бы очень удобно" (Настя, интервью #4)

## Сегмент 5: Бизнес/партнёр (подтверждено интервью #3)

| Поле | Описание |
|------|----------|
| **Customer Type** | B2B (представители бизнеса: технологические компании, потенциальные заказчики, инвесторы) |
| **Роли** | Ищут технологии/решения (основная), найм (вторичная), инвестиции (редко), партнёрство |
| **Демография** | 30-50 лет, руководители / техлиды / HR / инвестиционные аналитики |
| **Размер аудитории** | ~15-20 на DD (Олег: "пятнадцать-двадцать проектов, где я могу быть уверен") |
| **География** | Москва, крупные tech-компании |
| **Психография** | Ищут конкретные решения под бизнес-задачи. Не хотят "технический доклад, я ничего не понимаю". Готовы к follow-up, но нужна структурированная информация |
| **Поведение** | Приходят через личный нетворк Олега. Олег вручную подбирает комнату и проекты для каждого — бутылочное горлышко. Отправляют делегатов в конкретные комнаты. Post-DD запрашивают нарезки из Zoom, презентации |
| **Потребности** | Профилирование по бизнес-задаче → подборка проектов. Q&A-помощник с контекстом (Олег: "сто процентов не в контексте"). Follow-up с контактами авторов и презентациями |
| **Триггеры** | Личное приглашение от Олега/руководства Хаба, нетворкинг |
| **Процесс принятия решений** | Через Олега как посредника. Решение направить делегата — по конкретной бизнес-задаче |
| **Бюджет** | Не платят за DD. Ценность = найденные проекты/кадры/технологии |
| **Каналы** | Telegram (через Олега), личные встречи. Бот удобен для follow-up |
| **Возражения** | "Не находит свою проблематику", "технический доклад — отключаюсь" |
| **Как преодолеть** | Профилирование по бизнес-задаче (не по технологии), бот с Q&A-контекстом, бизнес-фокус в описаниях проектов |

**Ключевая цитата:** "Представитель бизнеса может описать свои боли и получить топ команд." — Олег Шатов (dream-сценарий)
