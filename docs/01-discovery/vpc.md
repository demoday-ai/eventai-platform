## Сегмент 1: Организатор Demo Day

### Customer Profile

#### Customer Jobs

**Функциональные:**
- Составить расписание 330 проектов по 6-10 залам
- Кластеризовать проекты по тематикам (NLP, Agents, CV, FinTech и др.)
- Набрать и распределить 60-80 экспертов по залам
- Обеспечить подтверждение участия студентов и экспертов
- Собрать и обработать обратную связь от экспертов
- Отслеживать доходимость в день DD

**Социальные:**
- Провести Demo Day без хаоса (репутация перед руководством и студентами)
- Показать профессиональную организацию внешним гостям и партнёрам
- Не подвести коллег-модераторов (обеспечить экспертов в каждый зал)

**Эмоциональные:**
- Не выгореть от ручной рутины ("этот ужас")
- Чувствовать контроль над мероприятием (вместо "хаос" и "пипец")
- Гордиться качеством мероприятия (вместо "с каждым разом хуже")

#### Pains
- Один человек собирает расписание вручную — single point of failure
- ChatGPT кластеризует плохо, дубликаты не ловятся
- 80 студентов не заполнили форму вовремя — экстренное распределение в ночь
- 15 проектов просто не пришли — нет системы подтверждения
- Нужно 60-80 экспертов, есть ~50, активных ~30-40
- Раньше эксперты выбирали комнаты сами → прыгали и не оценивали
- ОС от экспертов неконструктивная ("Херня!"), нельзя рассылать как есть
- Нет дашборда — не знают, кто пришёл, где пустые слоты
- Demo-Hero — "фигня", "максимально фиговый"

#### Gains
- Расписание составляется за часы, а не дни
- 0 дубликатов, 100% студентов распределены
- Подтверждение участия за 2 недели до DD
- Покрытие ≥90% залов экспертами
- Обратная связь доходит до студентов в тот же день
- Реальная картина: кто пришёл, кто нет, покрытие — в одном месте
- Не нужно лично писать каждому эксперту

---

### Value Map

#### Products & Services
- **AI-кластеризация проектов** — на основе описаний + GitHub + артефактов (не одной строчки и не ChatGPT)
- **Бот подтверждения участия** — персонально студентам и экспертам с эскалацией
- **Система матчинга экспертов** — по тегам интересов с подтверждением через бота
- **Рассылка напоминалок** — за день, за час, всем ролям
- **Предобработка ОС** — AI фильтрует неконструктивное, группирует по темам
- **Дашборд покрытия** — кто пришёл, пустые слоты, алерты
- **Единый Telegram-бот** с ролями (студент / эксперт / модератор / организатор / гость)

#### Pain Relievers
- Автокластеризация заменяет ChatGPT → 0 дубликатов, учёт GitHub, балансировка залов
- Персональные уведомления студентам заменяют массовые пинги → 80 человек узнают свой слот заранее
- Автоматические напоминалки заменяют ручные сообщения → неявка снижается
- Матчинг по тегам заменяет ручное распределение экспертов → каждый эксперт на своей теме
- AI предобработка ОС решает проблему "Херня!" → можно рассылать без ручной модерации каждого комментария
- Дашборд даёт реальную картину вместо "ходим по залам и спрашиваем"

#### Gain Creators
- Расписание формируется автоматически за часы → орг занимается стратегией, а не таблицами
- Подтверждение за 2 недели → есть время перераспределить слоты
- Покрытие экспертами видно на дашборде → можно добрать заранее
- Аналитика после DD: доходимость, NPS, качество ОС → insights для следующего DD
- Единый бот для всех ролей → нет разрозненных инструментов

## Сегмент 2: Эксперт / ментор

### Customer Profile

#### Customer Jobs

**Функциональные:**
- Оценить проекты на DD (в оценочных листах)
- Задать содержательные вопросы студентам
- Поставить справедливые оценки

**Социальные:**
- Внести вклад как ментор, помочь студентам
- Поддержать репутацию в профессиональном сообществе

**Эмоциональные:**
- Не чувствовать, что потратил время зря
- Получить интеллектуальное удовольствие от интересных проектов

#### Pains
- Идёт на защиты вслепую — не знает тематику и уровень проектов
- Попадает на нерелевантную комнату (техническому — продуктовые проекты и наоборот)
- Забывает/игнорирует оценки после DD
- "Презентация офигенная, а там вообще ничего нет" — нет способа проверить заранее
- Нет напоминалок, сложный процесс оценки через таблицу

#### Gains
- Знать заранее, какие проекты будут (карточки за день)
- Быть на теме, которая интересна (матчинг по тегам)
- Быстро оценить проекты (2 мин в Telegram, а не в таблице)
- Легко связаться с автором понравившегося проекта

---

### Value Map

#### Products & Services
- **Овервью проектов** — карточки за 24 часа до DD: описание, стек, GitHub, артефакты
- **Матчинг по тегам** — бот предлагает комнату по интересам, эксперт подтверждает
- **Быстрая оценка в Telegram** — бот присылает список проектов, оценка в 2 клика
- **Напоминалки** — за день, за час
- **Контакт с автором** — бот связывает по запросу

#### Pain Relievers
- Овервью заранее → не идёт вслепую, знает что ожидать
- Матчинг по тегам → попадает на релевантные проекты
- Оценка в Telegram → не нужно открывать таблицу, быстро и удобно
- Напоминалки → не забывает оценить, организаторам не нужно дёргать

#### Gain Creators
- GitHub-статус в карточке → видит, есть ли реальный код или "всё придумано"
- Подготовленные вопросы → более содержательное Q&A (но без AI-подсказок — сам решает)
- Быстрая связь с автором → нетворкинг без организатора-посредника

## Сегмент 3: Студент

### Customer Profile

#### Customer Jobs

**Функциональные:**
- Выступить на DD (часть учебного процесса)
- Получить оценку от экспертов
- Получить обратную связь для улучшения проекта

**Социальные:**
- Показать результат работы за семестр
- Получить признание от экспертов и однокурсников

**Эмоциональные:**
- Не волноваться о расписании (знать свой слот)
- Чувствовать, что обратная связь справедлива
- Понимать, как улучшить проект

#### Pains
- Не знает свой слот / не попал в расписание (80 чел.)
- Не получает обратную связь после DD
- Расписание публикуется поздно (ночь перед DD)
- Непрозрачная оценка: если эксперта не было — несправедливо
- Не понимает, как подготовиться к выступлению

#### Gains
- Знать слот за 2 недели
- Получить ОС в тот же день
- Структурированная ОС: техническое / продуктовое / презентация
- Уверенность, что эксперт будет на защите

---

### Value Map

#### Products & Services
- **Персональное уведомление о слоте** — "Ты выступаешь [дата, время, зал]" + подтверждение
- **Напоминалки** — за неделю, за день, за час
- **Рассылка ОС** — предобработанная, структурированная, в Telegram
- **Контакт с экспертом** — если эксперт хочет связаться

#### Pain Relievers
- Персональное уведомление → не нужно искать себя в расписании (80 чел. проблема решена)
- Подтверждение → оргам ясно, кто придёт, можно перераспределить
- Предобработанная ОС → получает конструктивный фидбэк без "Херня!"
- Гарантированный эксперт → каждый зал покрыт (через матчинг)

#### Gain Creators
- ОС в тот же день → сразу может работать над улучшениями
- Структура ОС (техническое / продуктовое / презентация) → понимает, что именно улучшать
- Напоминалки → помогает организовать подготовку
- Связь с экспертом → менторство и нетворкинг после DD

## Сегмент 4: Внешний гость (подтверждено интервью #4)

### Customer Profile

#### Customer Jobs

**Функциональные:**
- Найти релевантные проекты среди 330 по своим интересам (NLP, CV, антифрод и т.д.)
- Не пропустить самое интересное из-за сдвигов тайминга
- Получить конспект и follow-up после DD

**Социальные:**
- Быть в курсе AI-трендов, познакомиться с проектами перед поступлением/работой
- Задать содержательные вопросы авторам

**Эмоциональные:**
- Не чувствовать, что зря потратил время
- Не теряться среди "гигантской гугл-таблицы с миллионом вкладок"

#### Pains
- Расписание — "гигантская гугл-таблица, в которой чёрт ногу сломит" (Настя)
- Пропускает самое интересное из-за сдвигов тайминга: "не успела посмотреть один, самый интересный про фрод"
- Видит максимум 16% проектов при 6 параллельных залах
- Demo-Hero не помог
- Нет follow-up после мероприятия

#### Gains
- Персональная подборка проектов по интересам (5/5)
- Уведомления о сдвигах тайминга и напоминания за 5 мин
- Подсказки вопросов (3/5, "с поддержкой побольше задавала")
- Follow-up конспект (5/5)
- Канал: Telegram-бот, взаимодействие за неделю до DD + во время
- Время на профилирование: 5-10 мин максимум

---

### Value Map

#### Products & Services
- **Профилирование** — варианты + свободный текст ("поклацать"), 5-10 мин
- **Персональная программа** — подборка проектов под интересы
- **Уведомления** — напоминания за 5 мин до начала, алерты о сдвигах тайминга
- **Подсказки вопросов** — 3-5 вопросов под профиль гостя
- **Follow-up пакет** — конспект просмотренных проектов, рекомендации

#### Pain Relievers
- AI-профилирование заменяет "выбирай наугад" и "чёрт ногу сломит" в таблице
- Уведомления о сдвигах тайминга → не пропустит интересное
- Подборка учитывает параллельность залов → оптимальный маршрут

#### Gain Creators
- Персональная программа → видит самое релевантное (Настя: 5/5)
- Follow-up конспект → не нужно всё запоминать (5/5)
- Подсказки вопросов → увереннее задаёт вопросы авторам (3/5)
- Один бот для всего — "да, одно окно"

## Сегмент 5: Бизнес/партнёр (подтверждено интервью #3)

### Customer Profile

#### Customer Jobs

**Функциональные:**
- Найти проекты под бизнес-задачу (технология / найм / партнёрство / инвестиции)
- Оценить уровень проектов и команд
- Получить контакты авторов и презентации после DD

**Социальные:**
- Найти перспективные проекты для компании
- Показать руководству результаты визита (конкретные контакты, next steps)

**Эмоциональные:**
- Не "отключиться" от технического доклада без бизнес-контекста
- Чувствовать, что визит был продуктивным

#### Pains
- "Технический доклад, я ничего не понимаю, отключаюсь" (Олег)
- Олег — бутылочное горлышко: вручную подбирает проекты для каждого партнёра
- "Каннибализируем свой собственный трафик" — крутые проекты незамечены
- "Сто процентов не в контексте полностью, не могу ответить" — нет Q&A-помощника
- Post-DD: запрашивают нарезки из Zoom, презентации — делается вручную

#### Gains
- Профилирование по бизнес-задаче → подборка с бизнес-фокусом
- Q&A-помощник с контекстом проектов
- Follow-up: контакты авторов (с согласия), презентации, next steps, шаблоны LOI

---

### Value Map

#### Products & Services
- **Бизнес-профилирование** — задача + отрасль/стек → подборка проектов
- **Q&A-помощник** — AI с контекстом из репо/презентаций отвечает на вопросы бизнеса
- **Матрица сравнения** — проекты под бизнес-критерии (стадия, стек, команда)
- **Follow-up пакет** — контакты авторов (с согласия), нарезки презентаций, next steps, шаблоны LOI

#### Pain Relievers
- Бизнес-профилирование заменяет ручной подбор Олега → масштабируется
- Q&A-помощник заменяет Олега как "ходячую энциклопедию" → партнёр автономен
- Автоматический follow-up → не нужно вручную собирать материалы

#### Gain Creators
- Бизнес-подборка → партнёр находит релевантные проекты без Олега
- Матрица сравнения → можно показать руководству структурированные данные
- Шаблоны LOI → конвертирует DD в бизнес-результат

## Fit-анализ (Product-Market Fit по сегментам)

| Сегмент | Fit | Обоснование |
|---------|-----|-------------|
| **Организатор** | **Сильный** | Все боли подтверждены прямыми цитатами (интервью #1). Организатор описала часть решений как желаемые |
| **Эксперт** | **Средний-сильный** | Боли подтверждены (интервью #2). AI-вопросы отвергнуты, но овервью, матчинг и выбор слотов — да |
| **Студент** | **Средний** | Боли очевидны, но сами студенты не интервьюировались. "Студентам пофиг" → вовлечённость под вопросом |
| **Внешний гость** | **Сильный** | Подтверждено интервью #4 (Настя): персональная программа 5/5, follow-up 5/5, вопросы 3/5. Встречи 1:1 — 1/5 |
| **Бизнес/партнёр** | **Сильный** | Подтверждено интервью #3 (Олег): ручной подбор = бутылочное горлышко, Q&A-помощник критичен, follow-up уже делают вручную |
